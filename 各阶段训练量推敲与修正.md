# 各阶段训练量设置的推敲与修正建议

## 一、诚实评估：这些数字有多可靠？

### Stage 1：95k = 200 epoch ✅ **高度可靠**

**依据：**
- ✅ 文献充分：SSL4EO baseline 100 epoch，SatMAE 200 epoch
- ✅ 已完成验证：EuroSAT 69.57%，有实际效果数据
- ✅ 顶会共识：MAE 类方法 100-200 epoch 是标准区间

**结论：不需要改动**

---

### Stage 1.5：30k = 63 epoch ⚠️ **中等可靠，需实验验证**

**依据：**
- ✅ 续训比例：31.6% 符合文献经验（30-50%）
- ❓ 但缺乏：
  - 你自己数据的 pilot 实验
  - FiLM 模块的收敛速度实测
  - 对比学习（InfoNCE）的难度评估

**问题：**
1. **新模块参数少（2.9M）但任务抽象**
   - FiLM 只是缩放/偏移，收敛可能很快（15-20k 够？）
   - 但"学习成像无关表征"很抽象，可能需要更多样本

2. **对比学习的难度未知**
   - 季节对比：同地点不同季节（正例）vs 不同地点（负例）
   - 如果季节差异太小 → 难学习，需要更多 epoch
   - 如果季节差异明显 → 容易学，20k 可能就够

**修正建议：**

```python
# 渐进式训练策略（推荐）
Stage 1.5 分三阶段：
  
  Phase 1 (0-10k steps):
    - 只训 FiLM，冻结 encoder
    - w_decouple = 0.1（warm up）
    - 看 FiLM identity test 是否收敛
    → 如果 10k 已收敛，Phase 2 减半
  
  Phase 2 (10k-20k):
    - FiLM + 对比学习
    - w_decouple = 0.5
    - 监控 InfoNCE loss
    → 如果 20k 已平，可提前停
  
  Phase 3 (20k-30k):
    - 全解冻（可选）
    - w_decouple = 1.0
    - 精调所有模块
    → 如果验证集 loss 不降了，停
```

**保守估计：20-35k（根据收敛情况动态）**

---

### Stage 2：20k = 80 epoch ❌ **可靠性低，很可能不够**

**问题：**

1. **对比 EarthNet baseline（100-150 epoch）偏少**
   - 官方推荐：100-150 epoch
   - 我们：80 epoch（仅 baseline 的 53-80%）

2. **任务复杂度被低估**
   - 时序建模 >> 重建任务
   - 新增 dgh 注入、不确定性、rollout
   - 冻结编码器 → dynamics 模块压力更大

3. **数据集小但任务难**
   - EarthNet mini 仅 32k 样本（vs SSL4EO 244k）
   - 但时序依赖更复杂（需要学习动力学规律）
   - 小数据 + 复杂任务 → **需要更多 epoch 充分学习**

**修正建议：**

```python
Stage 2 应该至少 25-30k steps (100-120 epoch)

分三阶段（冻结 → 部分解冻 → 全解冻）:
  
  Stage 2a (0-15k): 冻结 encoder
    - 只训 dynamics 模块
    - 学习基本的状态转移
    - 监控 1-step prediction error
    → 如果 1-step 都不行，这阶段可能要 20k
  
  Stage 2b (15k-25k): 解冻最后几层 encoder
    - encoder 最后 2-3 层微调
    - dynamics 继续训练
    - 监控 multi-step rollout error
    → 关键阶段，可能需要延长
  
  Stage 2c (25k-30k): 全解冻（可选）
    - 联合优化
    - 小 lr（1e-5）
    → 如果 2b 已经很好，可跳过
```

**保守估计：25-35k（很可能需要上限）**

---

### Stage 3：15k = 60 epoch ✅ **较可靠，但可能偏保守**

**依据：**
- ✅ 解码任务相对简单（条件生成）
- ✅ 网络轻量（5-10M 参数）
- ✅ 60 epoch 符合 VQGAN decoder 经验

**但需注意：**
- 如果不用扩散模型，60 epoch 够
- 如果要高质量生成（PSNR > 30），可能需要 80-100 epoch
- 如果加入对抗训练（GAN），需要 100-150 epoch

**修正建议：**

```python
Stage 3 基础配置：15k (60 epoch)

可选扩展：
  - 如果 PSNR < 25 at 15k → 继续到 20k (80 epoch)
  - 如果要 perceptual quality → 加 GAN，延长到 30k (120 epoch)
  - 如果只要能看出大致形态 → 10k (40 epoch) 可能就够
```

**保守估计：15-25k（根据质量要求）**

## 三、实施策略：动态调整（强烈推荐）

### 核心思路：checkpoint + 验证集监控

```python
# Stage 1.5 实施
target_max = 30000
checkpoints = [10000, 20000, 30000]

for ckpt in checkpoints:
    train_to(ckpt)
    val_metrics = evaluate_validation()
    
    # 决策逻辑
    if ckpt == 10000:
        if val_metrics['film_identity'] < 0.01:
            print("✅ FiLM 已收敛，可提前进 Phase 2")
    
    if ckpt == 20000:
        if val_metrics['infonce_loss'] 已平:
            print("✅ 对比学习收敛，可以停了")
            break
        elif val_metrics['infonce_loss'] 还在降:
            print("⚠️ 还在学习，继续到 30k")
            continue
    
    if ckpt == 30000:
        print("✅ 完成 Stage 1.5")
```

### 关键指标

| 阶段 | 监控指标 | 停止条件 |
|------|---------|---------|
| Stage 1.5 | InfoNCE loss, FiLM identity error | 验证集 loss 连续 5k steps 不降 |
| Stage 2 | 1-step error, 5-step rollout error | Rollout error 不降且 < baseline |
| Stage 3 | PSNR, SSIM, perceptual loss | PSNR 不涨或达到目标（如 28dB）|

## 六、最关键的建议

**❗ 不要迷信固定的 step 数，要看验证集！**

1. **每 5k steps 保存 checkpoint**
2. **每 5k steps 跑验证集**
3. **监控关键指标的趋势**
4. **如果平了就停，如果还在降就继续**

**论文里怎么写：**
```
We train Stage X for N epochs with early stopping based on 
validation set performance. Training is stopped when the 
validation loss plateaus for 5k steps.
```

这样既科学，又能防止过训练/欠训练。
