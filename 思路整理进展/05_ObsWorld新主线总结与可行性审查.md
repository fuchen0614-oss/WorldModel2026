# ObsWorld 新主线总结与可行性审查

## 一、阶段性结论

当前更合理的主线不再是纯粹的 `WorldRS`，也不应退回单纯的像素未来帧预测。更稳、更贴近当前偏好的主线应表述为：

**ObsWorld：从有偏像素观测学习成像无关地表状态动力学的遥感世界模型。**

这条主线的核心是：

1. **世界不是遥感图像流，而是地表状态。**
2. **遥感图像是地表状态在特定成像条件下的有偏观测。**
3. **遥感 world model 应同时建模地表状态、状态动力学、观测模型。**
4. **像素预测必须保留，但它是观测模型的出口，不是世界模型的本体。**
5. **外生驱动状态转移保留为关键实验场，用来证明模型不是只做成像无关表征，而是真的能推演地表状态变化。**

一句话版本：

> 本文提出一个面向地表的遥感世界模型，从多源有偏像素观测中估计成像无关的地表状态，在该状态空间中建模外生驱动下的状态演化，并在给定未来成像条件下生成可验证的未来像素观测。

这相比 `output/01-04` 的变化是：旧方案把 `WorldRS` 的外生驱动状态转移作为第一主线，把 `ObsWorld v5` 的状态—观测分离作为支撑；新方案把 `ObsWorld v5` 的状态—动力学—观测三柱结构作为第一主线，把 `WorldRS` 的外生驱动状态转移作为最重要的验证场景。

## 二、该思路的合理性

### 2.1 为什么这个故事讲得通

遥感与自然视频不同。自然视频中的图像变化往往与相机运动、物体运动、交互动作相关；遥感中的图像变化则同时来自两类因素：

| 变化来源     | 例子                           | 对论文叙事的意义           |
| -------- | ---------------------------- | ------------------ |
| 地表真实状态变化 | 洪水淹没、城市扩张、作物生长、灾害损毁、土地覆盖转移   | 这是世界状态动力学，应由状态模型解释 |
| 成像观测条件变化 | 云、太阳角、传感器、SAR/光学差异、季节反射率、分辨率 | 这是观测偏置，应由观测模型解释    |

如果模型直接学习：

```text
过去图像 -> 未来图像
```

它很容易把“世界变了”和“拍摄方式变了”混在一起。新主线改为：

```text
过去有偏观测 + 成像条件 -> 成像无关地表状态
成像无关地表状态 + 外生驱动 + 地理先验 -> 未来地表状态
未来地表状态 + 未来成像条件 -> 未来像素观测
```

这样，像素预测不再是普通视频预测，而是对状态—观测分离结构的验证。

### 2.2 为什么仍然能叫 world model

不同领域对 world model 的定义并不完全一致，但共性通常包括：

1. 从观测中形成内部状态；
2. 学习状态随时间、条件或动作变化的规律；
3. 能在内部进行未来推演；
4. 能根据条件变化产生不同预测；
5. 评估不只看外观，还看一致性、可控性、任务有效性或约束合理性。

新主线对应如下：

| world model 共性 | ObsWorld 中的对应物 |
|---|---|
| 内部状态 | 成像无关地表状态 `s_t` 或信念 `b(s_t)` |
| 动力学 | `p(s_{t+h} | s_t, D, G, h)` |
| 条件响应 | 外生驱动 `D`、地理物理先验 `G`、成像条件 `phi` |
| 未来推演 | 未来状态图、状态转移图、未来像素观测 |
| 可验证性 | 像素指标、状态转移指标、驱动敏感性、物理一致性、不确定性校准 |

因此，它不是“给遥感模型贴 world model 标签”，而是把 world model 的状态空间结构迁移到遥感语境中。

### 2.3 为什么像素预测应该保留

完全不做像素预测会带来三个问题：

1. 很难和现有未来预测、生成模型、遥感世界模型工作公平对比；
2. 结果不直观，审稿人可能质疑潜状态是否真的有用；
3. `ObsWorld v5` 的观测模型如果没有像素出口，就难以证明“状态如何被观测”。

但像素预测必须被放在正确位置：

```text
不推荐：我们预测未来像素，所以我们是世界模型。
推荐：我们预测地表状态如何演化，并通过观测模型生成在指定成像条件下会看到的未来像素。
```

这使像素预测成为优势，而不是把论文拖入纯生成模型红海。

## 三、可行性审查

### 3.1 数据可行性

当前没有自有数据集，但公开数据足够支撑第一版论文。关键是不要寻找一个“天然包含所有字段的完美数据集”，而是用统一 schema 重构不同公开数据。

| 数据集 | 主要提供什么 | 在本文中的作用 | 主要限制 |
|---|---|---|---|
| [SSL4EO-S12 v1.1](https://github.com/DLR-MF-DAS/SSL4EO-S12-v1.1) | 246,144 个地点、四个时间戳、S1/S2、NDVI、LULC、DEM | 从零自监督预训练，多模态、多季节、粗状态、地形先验 | 不包含强事件驱动，不能单独支撑洪水/灾害动力学 |
| [EarthNet2021](https://arxiv.org/abs/2104.10066) | Sentinel-2、地形、天气变量、32,000 个样本 | 未来像素预测主任务，支撑“像素观测出口” | 主要是光学，状态标签较弱 |
| [DynamicEarthNet](https://arxiv.org/abs/2203.12560) | 75 个 AOI、每日 Planet 影像、月度 7 类 LULC 标签 | 季节/土地覆盖状态变化，未来状态图与状态转移图 | Planet 许可与数据获取需要谨慎 |
| [SEN12-FLOOD](https://source.coop/esa/sen12flood) | 配准的 Sentinel-1/2 洪水时序与标签 | 洪水状态转移，多模态观测偏置，SAR/光学互补 | 事件数量和驱动强度字段有限 |
| [Sen1Floods11](https://openaccess.thecvf.com/content_CVPRW_2020/html/w11/Bonafilia_Sen1Floods11_A_Georeferenced_Dataset_to_Train_and_Test_Deep_Learning_CVPRW_2020_paper.html) | 4,831 个 512×512 chips，11 次洪水事件 | 洪水分割与永久水体/洪水水体状态转移评估 | 原生历史序列不足，需要外挂前期观测或只做冲击响应 |
| [SpaceNet7](https://arxiv.org/abs/2102.11958) | 多月 Planet 影像与建筑 footprint | 城市扩张慢变量状态转移 | 驱动条件较弱，容易像变化检测 |
| [xBD](https://arxiv.org/abs/1911.09296) | 灾前/灾后影像、19 类灾害、850,736 建筑 polygon | 事件冲击响应、建筑损毁状态转移 | 单步，不是连续时序；高分影像许可需注意 |

结论：

```text
公开数据足够支撑中等规模 ObsWorld 从零训练与多任务验证；
公开数据不足以支撑“复刻 SkySense 规模和通用性”的叙事。
```

### 3.1.1 数据使用逻辑：不是一股脑全部丢进去

需要特别区分三种数据角色：

```text
预训练数据：让模型学遥感观测的通用规律
能力训练数据：让模型学会按 world model 的接口输入输出
下游评估数据：证明模型在具体任务上有用
```

因此，大模型从零训练不是把 `SSL4EO-S12 v1.1`、`EarthNet2021`、`DynamicEarthNet`、洪水、城市、灾害数据全部无脑混在一起。更合理的顺序是：

```text
阶段一：SSL4EO-S12 v1.1 做从零自监督预训练
目标：学 S1/S2、多季节、多模态、地形和粗状态表征

阶段二：EarthNet2021 + DynamicEarthNet + 洪水数据做世界模型能力训练
目标：学未来像素预测、未来状态预测、状态转移预测

阶段三：下游微调/评估
目标：和遥感基础模型、未来预测模型、任务专家模型公平比较
```

后两个阶段可以多任务混合，但必须满足三个条件：

1. 每个样本带 `task_id`；
2. 每个字段带 `mask`，没有的字段不能强行填假真值；
3. 每个任务只计算自己有监督信号的 loss。

也就是说，真正的训练组织方式应是：

```text
共享观测编码器和状态空间；
不同任务使用统一 schema；
不同任务激活不同输入字段、输出头和损失函数。
```

如果无脑混合，模型会同时收到不一致的监督：有的样本要求重建未来像素，有的样本要求预测土地覆盖，有的样本要求预测洪水，有的样本要求预测建筑损毁。这样会使论文变成“多任务工程拼盘”，而不是一个清晰的遥感 world model。

### 3.1.2 需要补充的主要不是新影像，而是字段

当前没有自有数据集并不等于不能做。更准确地说，本文需要补充的多数不是“新遥感影像”，而是把公开数据补齐成 world model 所需的字段。

理想统一输入是：

```text
过去影像 + 当前状态图 + 外生驱动 + 地理物理先验 + 未来成像条件
```

公开数据通常不会天然给齐这些字段，所以需要按数据集补充：

| 数据集 | 原本较强的字段 | 需要补充的字段 | 补充目的 |
|---|---|---|---|
| SSL4EO-S12 v1.1 | S1/S2、四季、DEM、LULC、NDVI | 成像条件 `phi`、月份/季节、云状态、可选弱状态标签 | 用于预训练状态—观测关系 |
| EarthNet2021 | 过去/未来 S2、天气、地形 | 粗状态图、未来成像条件、可选 LULC 辅助标签 | 让像素预测变成状态驱动的观测预测 |
| DynamicEarthNet | 日影像、月度 LULC 标签 | DEM、气候带、月份/季节、可选天气 | 构造 `S_t -> S_{t+h}` 状态转移 |
| SEN12-FLOOD | S1/S2 洪水影像/标签 | DEM、坡度、永久水体、河流距离、事件/降雨字段 | 支撑洪水驱动与物理一致性 |
| Sen1Floods11 | 洪水期 S1 与洪水标签 | 洪水前影像、永久水体、DEM、河流/降雨 | 区分“本来就是水”和“新被淹” |
| SpaceNet7 | 月度影像、建筑 footprint | OSM 道路、永久水体、历史增长强度 | 构造城市扩张状态转移 |
| xBD | 灾前/灾后影像、损毁等级 | 灾害强度、DEM、建筑先验、事件元数据 | 构造灾害冲击响应 |

因此，所谓“构建数据集”更像是：

```text
下载公开数据
统一 AOI / 时间 / 坐标
补充公开地理图层
补充事件、季节、成像条件字段
派生当前状态、未来状态、状态转移
输出统一 manifest 与 dataloader
```

这不同于从零采集一个新的大规模遥感影像库。

### 3.2 从零训练是否可行

可行，但要控制表述。

推荐表述：

> 我们在公开多源时序遥感数据上从零训练一个中等规模的遥感世界模型，用于验证状态—观测分离和条件动力学结构。

不推荐表述：

> 我们从零训练一个超越现有遥感基础模型的通用大模型。

原因是现有强模型规模很大。例如 [SkySense](https://arxiv.org/abs/2312.10115) 是 2.06B 参数，使用 21.5M 多模态时序序列；[Prithvi-EO-2.0](https://github.com/NASA-IMPACT/Prithvi-EO-2.0) 使用 4.2M HLS 全球时序样本，并有 300M/600M 版本。本文不应与它们拼规模，而应证明结构性能力。

### 3.3 成像条件学习是否足够

只用 Sentinel-1/2 时，视角变化确实有限。如果把 `phi` 只定义成太阳角/视角，证据会偏弱。

建议把成像条件 `phi` 定义为更宽的观测条件：

```text
phi = 传感器/模态 + 时间/月份/季节 + 云状态 + 太阳角 + SAR 极化/入射角 + 分辨率/产品级别
```

最强的观测偏置不是单一视角，而是：

1. 光学与 SAR 成像机理差异；
2. 晴天与云遮挡差异；
3. 季节导致的反射率/植被物候差异；
4. Sentinel、Landsat/HLS、Planet 等传感器差异；
5. 太阳高度角与时间差异。

第一版可以主要依赖 S1/S2 模态差异、季节差异和云条件。若时间允许，再引入 HLS/Landsat 或 Planet 数据增强传感器差异。

### 3.4 地理物理先验是否可行

不建议把主线建立在纯地学模拟器生成成对数据上。更稳的是使用公开地理图层作为输入或验证器：

| 先验 | 构造方式 | 用途 |
|---|---|---|
| DEM | SSL4EO 自带或 Copernicus/SRTM 裁剪 | 高程、坡度、洪水合理性 |
| 坡度/坡向 | DEM 派生 | 洪水/城市扩张约束 |
| 永久水体 | JRC Global Surface Water 或类似水体产品 | 区分永久水体和新增洪水 |
| 河流距离 | HydroRIVERS 或 OSM 派生 | 洪水物理一致性 |
| 道路距离 | OpenStreetMap 派生 | 城市扩张与建筑合理性 |
| 当前土地覆盖 | LULC、ESA WorldCover、Dynamic World 或数据集标签 | 当前状态先验 |
| 月份/气候带 | 时间戳、Koppen/ERA5 派生 | 季节状态转移 |

推荐构造方式：

```text
样本经纬度/AOI/时间 -> 批量裁剪公开地理图层 -> 重采样到影像网格 -> 存成本地缓存 -> 训练时直接读取
```

地学模拟器可作为辅助验证器或合成消融，不建议作为主训练数据来源。

## 四、AAAI 与 CVPR 匹配度

### 4.1 对 AAAI 的匹配度

AAAI 更可能接受这条线的原因：

1. 叙事具有 world model、状态空间、信念、条件动力学的 AI 味道；
2. 可以强调“遥感语境下 world model 的定义与评估协议”；
3. 可突出可解释性、不确定性、驱动敏感性和物理一致性；
4. 不必把视觉生成质量卷到极致。

AAAI 风险：

1. 如果方法结构过于工程化，会被认为只是多任务遥感框架；
2. 如果 world model 定义太松，会被认为概念包装；
3. 如果实验不能证明条件动力学和状态—观测分离的必要性，叙事会塌。

AAAI 推荐写法：

```text
提出遥感 world model 的状态空间定义；
提出成像无关状态—动力学—观测模型统一框架；
提出跨任务评估协议验证观测偏置分离、条件动力学和物理一致性。
```

### 4.2 对 CVPR 的匹配度

CVPR 更看重视觉方法、图像结果、强 baseline 和视觉直观性。新主线比旧主线更适合 CVPR 的地方在于：保留了未来像素预测作为观测出口，可以和图像预测/生成模型对比。

CVPR 需要重点补强：

1. 像素预测质量必须可看、可量化；
2. 跨模态/跨成像条件的可视化要清楚；
3. 与遥感基础模型、未来预测模型、生成模型的对比要完整；
4. 消融必须证明状态—观测分离确实提升像素预测、状态预测和泛化。

CVPR 风险：

1. 像素图质量弱于生成模型时，审稿人可能只盯视觉指标；
2. 半物理观测模型若实现复杂但增益不明显，会被认为过度设计；
3. 多数据集 schema 如果不清楚，会被看成工程整合。

CVPR 推荐写法：

```text
我们不是直接做未来帧生成，而是提出状态驱动的条件观测预测：
模型先预测地表状态如何演化，再在指定成像条件下生成未来遥感观测。
```

### 4.3 总判断

| 会议 | 匹配度 | 更需要证明什么 |
|---|---|---|
| AAAI | 高 | world model 定义清晰、状态空间合理、条件动力学可证伪 |
| CVPR | 中高 | 像素预测强、视觉结果清楚、与强 baseline 对比充分 |

当前主线若执行得好，AAAI 更自然，CVPR 也可冲。若像素预测质量和视觉展示足够强，CVPR 竞争力会上升；若像素质量一般但状态/不确定性/物理一致性很扎实，AAAI 更稳。

## 五、整体流程

### 5.1 训练与评估总流程

```text
阶段一：公开多源遥感数据自监督预训练
输入：多时相 S1/S2/HLS/Planet 影像、成像条件、DEM/LULC/NDVI 等弱辅助层
目标：学习可解释多源观测的基础表征与初始状态空间

阶段二：状态—观测分离训练
输入：历史观测 X_{t-k:t}、对应成像条件 phi_{t-k:t}
目标：估计成像无关状态 s_t，使其能解释不同模态/季节/云条件下的观测

阶段三：状态动力学与像素观测预测训练
输入：s_t、外生驱动 D、地理先验 G、预测跨度 h、未来成像条件 phi_{t+h}
输出：未来状态 S_{t+h}、状态转移 Delta S、未来像素 X_{t+h}

阶段四：多任务评估
评估：像素质量、状态转移、驱动敏感性、物理一致性、不确定性校准
```

### 5.2 模型模块流程

```text
历史像素观测 X_{t-k:t}
+ 历史成像条件 phi_{t-k:t}
        |
        v
观测编码器 f_obs
        |
        v
成像无关状态估计器 q(s_t | X, phi)
        |
        +-----------------------------+
        |                             |
        v                             v
状态动力学 T(s_t, D, G, h)       当前观测重建 g(s_t, phi_t)
        |
        v
未来状态 s_{t+h}
        |
        +-----------------------------+
        |                             |
        v                             v
状态/转移解码器                 观测模型 g(s_{t+h}, phi_{t+h})
        |                             |
        v                             v
未来状态图 S_{t+h}              未来像素图 X_{t+h}
状态转移图 Delta S              像素不确定性/观测一致性
```

### 5.3 输入输出总表

| 名称     | 符号            | 具体内容                            | 来源                                            | 是否必需   |
| ------ | ------------- | ------------------------------- | --------------------------------------------- | ------ |
| 历史影像   | `X_{t-k:t}`   | S1、S2、Planet、HLS、Landsat 等多时相影像 | SSL4EO、EarthNet、DynamicEarthNet、SEN12-FLOOD 等 | 必需     |
| 历史成像条件 | `phi_{t-k:t}` | 传感器、模态、时间、月份、云、太阳角、SAR 极化、入射角   | 元数据、时间经纬度反算、云掩膜                               | 必需     |
| 当前状态先验 | `S_t`         | 当前 LULC、水体/非水体、建筑、作物、灾前状态       | 标签、外部产品、模型估计                                  | 建议     |
| 外生驱动   | `D`           | 季节、天气、洪水事件、灾害类型、城市增长强度、时间跨度     | 数据集元信息、天气再分析、事件标签                             | 任务相关   |
| 地理物理先验 | `G`           | DEM、坡度、水体距离、道路距离、气候带、永久水体       | 公开地理图层                                        | 建议     |
| 未来成像条件 | `phi_{t+h}`   | 目标传感器/模态、未来月份、目标云态、天气条件         | 数据自带或任务设定                                     | 像素预测必需 |
| 预测跨度   | `h`           | 未来天数、月份、事件后间隔                   | 数据集时间戳                                        | 必需     |
| 未来像素图  | `X_{t+h}`     | 未来 S2/Planet/HLS 观测             | 监督标签或预测输出                                     | 主输出之一  |
| 未来状态图  | `S_{t+h}`     | 未来 LULC、水体、建筑、损毁等级等             | 标签或预测输出                                       | 主输出之一  |
| 状态转移图  | `Delta S`     | 每个像素/对象从 `S_t` 到 `S_{t+h}` 的变化  | 由 `S_t` 与 `S_{t+h}` 派生                        | 主输出之一  |
| 不确定性   | `U`           | 像素/状态/转移不确定性                    | 模型预测                                          | 建议     |

## 六、输入输出构建方法

### 6.1 历史影像构建

从数据集中按 AOI 和时间组织：

```text
X_{t-k:t} = [X_{t-k}, ..., X_t]
```

示例：

| 数据集 | 历史影像构建 |
|---|---|
| SSL4EO-S12 v1.1 | 四季 S1/S2 时间戳构成多时相输入 |
| EarthNet2021 | 过去若干 Sentinel-2 帧作为输入 |
| DynamicEarthNet | 日尺度 Planet 影像按月/季节采样 |
| SEN12-FLOOD | 洪水事件前后 S1/S2 序列 |
| SpaceNet7 | 多月 Planet 序列 |

### 6.2 当前状态构建

当前状态 `S_t` 可以有三种来源：

| 方法 | 说明 | 适用场景 |
|---|---|---|
| 真实标签 | 使用数据集当前时刻标签 | DynamicEarthNet、SpaceNet7、xBD |
| 外部产品 | 使用 LULC、JRC 水体、ESA WorldCover、Dynamic World | SSL4EO、洪水任务 |
| 模型估计 | 先由状态估计器从影像预测 `S_t` | 没有当前标签的任务 |

第一版建议不要强求所有任务都有高质量 `S_t` 标签。可以使用：

```text
显式语义状态 S_t + 连续潜状态 z_t
```

语义状态负责可评估，连续潜状态负责保留细粒度像素信息。

### 6.3 外生驱动构建

外生驱动不是每个数据集都完整提供，需要按任务派生：

| 任务 | 驱动字段 |
|---|---|
| 像素未来预测 | 未来天气、月份、季节、时间跨度 |
| 土地覆盖变化 | 源月、目标月、季节、气候带 |
| 洪水 | 洪水事件 ID、事件区域、事件时间、可选降雨/河流/水系 |
| 城市扩张 | 时间跨度、历史增长率、距离道路/已有建筑 |
| 灾害损毁 | 灾害类型、事件 ID、灾前/灾后间隔、可选震级/风速等 |

第一版可以允许弱驱动：

```text
D = 事件类型 + 时间跨度 + 月份/季节 + 区域上下文
```

未来若能批量外挂 ERA5、GPM 降雨、灾害事件数据库，再升级为强驱动。

### 6.4 地理物理先验构建

推荐使用“索引 + 脚本 + 缓存”模式：

```text
样本 AOI/经纬度/时间
    -> 批量裁剪 DEM、水体、道路、土地覆盖等图层
    -> 重采样到影像坐标
    -> 保存为本地缓存
    -> 训练/评估读取
```

不要训练时在线调用外部服务。公开发布时也不重分发受限原始影像，而发布：

1. 样本索引；
2. 下载与裁剪脚本；
3. 坐标对齐规则；
4. 派生字段说明；
5. schema 与 dataloader；
6. 评估协议。

### 6.5 未来状态图与状态转移图构建

未来状态图：

```text
S_{t+h}(p) = 像素或对象 p 在未来时刻的状态类别
```

状态转移图：

```text
Delta S(p) = (S_t(p), S_{t+h}(p))
```

示例：

| 任务 | 未来状态图 | 状态转移图 |
|---|---|---|
| 洪水 | 非水体、永久水体、洪水水体 | 非水体->洪水，永久水体->永久水体 |
| 土地覆盖 | 7 类 LULC | 农田->裸地，植被->建筑，水体->水体 |
| 城市扩张 | 建筑/非建筑 | 非建筑->建筑，建筑->建筑 |
| 灾害损毁 | 无损、轻损、重损、摧毁 | 建筑完好->不同损毁等级 |

## 七、推荐下游任务

### 7.1 第一版推荐组合

| 优先级 | 任务 | 数据集 | 证明什么 |
|---|---|---|---|
| 必做 1 | 未来像素预测 | EarthNet2021 | 像素观测出口、与未来预测模型对比 |
| 必做 2 | 季节/土地覆盖状态变化 | DynamicEarthNet | 状态图、状态转移、季节驱动 |
| 必做 3 | 洪水状态转移 | SEN12-FLOOD + Sen1Floods11 | 外生事件驱动、SAR/光学观测偏置、物理一致性 |
| 可选 4 | 城市扩张 | SpaceNet7 | 慢变量状态演化 |
| 可选 5 | 灾害损毁 | xBD | 事件冲击响应、对象级状态转移 |

### 7.2 不建议第一版全做

不建议一次性铺满五六个任务。原因：

1. 容易变成 benchmark engineering；
2. 统一 schema 难度高；
3. baseline 成本高；
4. 每个任务的故事深度会被摊薄。

更稳做法：

```text
三主任务 + 一扩展任务
```

其中像素预测、土地覆盖/季节、洪水必须优先，因为它们分别支撑“像素出口”“状态转移”“外生驱动”。

### 7.3 其他可选数据集与暂不优先选择的原因

公开遥感数据集还有很多，但第一版不能全部拼进去。选择数据集时应遵守一个原则：

```text
每个数据集必须服务一个明确论文能力；
不能只是为了显得任务多而加入。
```

可选数据集与判断如下：

| 数据集 | 可支持的能力 | 暂不作为第一优先的原因 |
|---|---|---|
| BigEarthNet | 大规模多标签场景理解、LULC 语义预训练 | 主要是静态分类/多标签，不直接提供未来状态或像素预测 |
| SEN12MS | S1/S2 多模态对齐、土地覆盖相关表征 | 更适合预训练或跨模态表征，不是状态转移主任务 |
| SSL4EO-L / HLS / Landsat 数据 | 更多传感器差异、长时序、成像条件扩展 | 数据整理成本高，可作为第二阶段增强，而不是第一版必要项 |
| PASTIS / PASTIS-R | 作物/物候、S1/S2 时间序列、季节状态变化 | 很适合补强季节与作物，但会新增农业专门任务，第一版可作为扩展 |
| BreizhCrops / TimeSen2Crop | 作物分类与物候时间序列 | 适合农业状态，不一定支撑像素未来预测 |
| LEVIR-CD / S2Looking | 建筑变化检测 | 变化检测很强，但外生驱动弱，容易把主线拉回普通变化检测 |
| C2S-MS Floods | 洪水多光谱相关任务 | 可补强洪水，但需核查标签、时序和许可，第一版可作为补充 |
| Dynamic World / ESA WorldCover | 全球 LULC 产品 | 更适合作为状态先验或伪标签，不适合单独作为主任务 |

为什么不把这些全部合并？

1. **监督目标不同**：有的是分类，有的是分割，有的是变化检测，有的是未来预测；
2. **空间分辨率不同**：10m、3m、30m、高分商业影像混在一起会增加对齐难度；
3. **时间结构不同**：有的单时相，有的双时相，有的长时序；
4. **许可不同**：Planet、Maxar、高分商业数据往往不能随意重分发；
5. **驱动字段不同**：很多变化检测数据只有前后图，没有明确外生驱动；
6. **baseline 成本不同**：每加一个任务就要补对应专家模型和评估协议。

因此，第一版推荐组合的逻辑是：

| 论文能力 | 首选数据 | 为什么选它 |
|---|---|---|
| 从零预训练 | SSL4EO-S12 v1.1 | 公开、多模态、多季节、带 DEM/LULC/NDVI，适合打地基 |
| 未来像素预测 | EarthNet2021 | 本身就是未来地表观测预测任务，方便和现有方法对比 |
| 季节/状态转移 | DynamicEarthNet | 有月度 LULC 标签，可构造未来状态图和转移图 |
| 外生事件驱动 | SEN12-FLOOD + Sen1Floods11 | 洪水是最清楚的外生驱动状态转移任务，且 SAR/光学差异强 |
| 扩展泛化 | SpaceNet7 或 xBD | 一个代表慢变量城市扩张，一个代表灾害冲击响应，二选一即可 |

换句话说，第一版不是追求“数据越多越好”，而是追求：

```text
每个数据集对应一个不可替代的叙事功能。
```

## 八、对比方法设计

### 8.1 对比对象类别

| 类别 | 代表方法 | 对比目的 |
|---|---|---|
| 遥感基础模型 | SkySense、Prithvi-EO-2.0、CROMA、DOFA、SatMAE、Scale-MAE | 证明不是普通表征迁移 |
| 像素未来预测模型 | EarthNet2021 baseline、ConvLSTM、PredRNN、Earthformer、UNet/3D-UNet | 比未来像素预测能力 |
| 遥感生成/世界模型 | RS-WorldModel、RemoteBAGEL、TerraMind 可用部分 | 切割未来场景生成与状态—观测分离 |
| 任务专家模型 | 洪水分割、土地覆盖变化、建筑变化、灾害损毁 SOTA | 证明实际任务性能够格 |
| 自身消融 | 无成像条件、无状态—观测分离、无动力学、无地理先验、无像素出口 | 证明每个模块服务主线 |

### 8.2 冻结/微调对比设置

对 SkySense、Prithvi 等模型，不应只做一种对比。建议至少三档：

| 设置 | 含义 | 回答的问题 |
|---|---|---|
| 冻结编码器 + 同样任务头 | 大模型权重不动，只训练分割/预测头 | 原始表征是否够好 |
| 轻量微调 | LoRA 或部分层微调 | 大模型适配当前任务后的能力 |
| 大模型编码器 + ObsWorld 机制 | 使用大模型作为 `f_obs`，接状态/动力学/观测模块 | ObsWorld 机制是否能迁移到强骨干 |

本文自己的模型也应在相同任务头下比较，避免被说“任务头不公平”。

### 8.3 不要追求所有常规指标第一

合理目标是：

```text
常规像素/任务指标：接近或部分超过强 baseline；
ObsWorld 专属指标：明显优于直接预测和普通基础模型。
```

专属指标包括：

1. 跨成像条件泛化；
2. 成像条件替换敏感性；
3. 状态转移准确性；
4. 驱动敏感性；
5. 物理一致性；
6. 不确定性校准；
7. 像素预测中结构保持与状态一致性。

## 九、实验设计

### 9.1 主实验

| 实验 | 数据 | 指标 | 证明点 |
|---|---|---|---|
| 未来像素预测 | EarthNet2021 | EarthNetScore、SSIM、PSNR、LPIPS、MAE | 观测出口有效，可与现有方法比 |
| 土地覆盖状态转移 | DynamicEarthNet | mIoU、变化 F1、转移混淆矩阵 | 状态动力学有效 |
| 洪水状态转移 | SEN12-FLOOD / Sen1Floods11 | IoU、F1、永久水体/洪水混淆 | 外生事件驱动有效 |
| 跨模态预测 | S1/S2 数据 | 模态互预测、跨模态状态一致性 | 成像无关状态有效 |

### 9.2 世界模型能力实验

| 实验 | 做法 | 证明点 |
|---|---|---|
| 驱动消融 | 去掉或置空 `D` | 模型是否真正使用外生驱动 |
| 错误驱动替换 | 洪水样本输入非洪水/城市驱动 | 模型响应是否下降或不确定性上升 |
| 未来成像条件替换 | 固定状态，替换未来模态/季节/云条件 | 观测模型是否可控 |
| 地理先验消融 | 去掉 DEM、水体距离、道路距离等 | 物理先验是否有贡献 |
| 状态—观测分离消融 | 直接像素预测，不经过状态空间 | 状态空间是否必要 |
| 状态一致性评估 | 从生成未来像素再分割出状态，与预测状态比 | 像素出口是否尊重状态 |
| 不确定性校准 | ECE、AUSE、错误-不确定性相关 | 模型是否知道何时不可信 |

### 9.3 可视化实验

建议必须有以下图：

1. 过去观测、预测未来像素、真实未来像素；
2. 当前状态图、未来状态图、状态转移图；
3. 不同未来成像条件下的观测输出；
4. 有/无外生驱动的预测差异；
5. 有/无地理先验的物理一致性差异；
6. 不确定性热图与错误区域对齐。

## 十、合理性与风险再判断

### 10.1 合理性

该方案合理，因为它解决了 `output/01-04` 中的两个矛盾：

1. 旧主线担心过度偏向潜状态、难以和像素预测模型比较；新主线保留未来像素观测出口。
2. 旧主线选择 WorldRS 为主，用户更喜欢 ObsWorld v5；新主线把 ObsWorld v5 的三柱结构放回中心。

### 10.2 最大风险

| 风险 | 严重程度 | 应对 |
|---|---|---|
| 像素预测质量不如生成模型 | 高 | 不只比 FID/SSIM，加入状态一致性和条件可控性 |
| 数据字段不完整 | 高 | 使用带 mask 的统一 schema，允许弱驱动与任务特定字段 |
| 成像条件证据不足 | 中高 | 扩大 `phi` 定义，强调模态/云/季节/传感器差异 |
| 多任务过宽 | 中高 | 第一版控制在三主任务一扩展 |
| 被看成工程拼接 | 高 | 所有模块必须围绕状态—动力学—观测模型三柱解释 |
| 与 RS-WorldModel 混淆 | 中高 | 明确本文不是文本引导未来场景生成，而是状态驱动的条件观测预测 |

## 十一、相较于 01-04 的改进

### 附录 A：主线重心的变化

`output/01-04` 的最终推荐是：

```text
外生驱动条件下的遥感地表状态转移世界模型
```

新方案调整为：

```text
从有偏像素观测学习成像无关地表状态动力学的遥感世界模型
```

核心变化是：由 `WorldRS` 主导转为 `ObsWorld v5` 主导。

### 附录 B：像素预测的位置变化

旧方案中像素预测被弱化为辅助解码和可视化，担心进入生成红海。

新方案保留这个警惕，但重新定义像素预测：

```text
像素预测不是世界模型本体；
像素预测是观测模型出口；
像素预测用于公平对比、可视化和验证状态是否能被观测。
```

这更符合当前偏好，也更方便与现有模型对比。

### 附录 C：WorldRS 的位置变化

旧方案中 `WorldRS` 是主线。

新方案中 `WorldRS` 变为关键实验场：

```text
外生驱动状态转移用于证明 ObsWorld 的状态空间具备动力学推演能力。
```

这样既保留了 WorldRS 的可证伪实验优势，又不牺牲 ObsWorld v5 的思想核心。

### 附录 D：数据可行性的细化

`output/01-04` 已提出多个数据集，但没有充分解释“理想输入字段如何从公开数据构建”。

本文件补充了：

1. SSL4EO-S12 v1.1 用于从零预训练；
2. EarthNet2021 用于未来像素预测；
3. DynamicEarthNet 用于季节/土地覆盖状态转移；
4. SEN12-FLOOD 与 Sen1Floods11 用于洪水驱动；
5. SpaceNet7/xBD 作为扩展任务；
6. 地理先验用公开图层裁剪，不依赖纯模拟器生成主数据。

### 附录 E：会议定位的细化

旧方案判断“AAAI/CVPR 都可”。

本文件更明确：

| 会议 | 新判断 |
|---|---|
| AAAI | 更自然，适合强调 world model 定义、状态空间、条件动力学、可信评估 |
| CVPR | 可冲，但必须让像素预测和视觉结果足够强，并补齐强 baseline |

### 附录 F：当前最新推荐

当前最建议推进的论文主线是：

**ObsWorld：从有偏像素观测学习成像无关地表状态动力学的遥感世界模型。**

推荐第一版任务组合：

```text
从零预训练：SSL4EO-S12 v1.1
主任务 1：EarthNet2021 未来像素预测
主任务 2：DynamicEarthNet 季节/土地覆盖状态转移
主任务 3：SEN12-FLOOD + Sen1Floods11 洪水状态转移
扩展任务：SpaceNet7 或 xBD 二选一
```

推荐贡献写法：

1. 提出遥感 world model 的状态—动力学—观测三柱定义；
2. 提出成像无关地表状态驱动的未来观测预测框架；
3. 构建公开数据上的统一训练/评估 schema；
4. 通过像素预测、状态转移、驱动敏感性、物理一致性和不确定性校准验证世界模型能力。
