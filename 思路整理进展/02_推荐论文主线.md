# 推荐论文主线

## 一、总推荐

我最推荐的论文主线是：

**面向像素观测的遥感地表状态动力学世界模型。**

这条主线的核心不是“生成未来遥感影像”，而是“从像素观测中估计地表状态，并建模该状态在外生驱动和地理物理约束下如何转移”。像素仍然重要，但它是状态的观测入口、监督来源和可视化出口，不是论文的最终哲学对象。

更聚焦的标题方向可以写成：

**外生驱动条件下的遥感地表状态转移世界模型。**

二者的区别是：前者更照顾你对像素空间的偏好，后者更稳、更像顶会论文主线。

## 三、推荐主线的论文一句话

**遥感世界模型不应只是生成未来影像，而应从有偏、稀疏、多源的像素观测中恢复地表状态，并学习该状态在外生驱动与地理物理约束下的可检验演化规律。**

这句话有三个好处：

1. 避开“只是视频预测”的风险；
2. 吸收 `ObsWorld` 的状态—观测分离；
3. 保留 `WorldRS` 的外生驱动与情景模拟能力。

| `WorldRS：外生驱动条件下的地表状态动力学世界模型` | 中高 | 名称清楚，但要避免与已有 RS-WorldModel 混淆。 |
| `像素空间遥感世界模型` | 低 | 容易被理解为像素级视频生成，不建议作为正式标题。 |
| `主动感知遥感世界模型` | 中 | 可作为后续方向，当前会分散主线。 |

## 五、Introduction 叙事骨架

### 第一段：现有遥感大模型的成绩与盲区

遥感基础模型、视觉语言模型和多模态模型已经能做分类、检测、分割、变化描述、多模态对齐和未来场景生成。例如 SkySense 代表多模态遥感基础模型，TerraMind 代表任意到任意多模态生成，RS-WorldModel 代表变化理解与未来场景预测。

但这些模型大多仍以“观测”为中心：理解观测、对齐观测、补全观测、生成观测。

### 第二段：关键矛盾

遥感里的图像变化不等于地表变化。云、太阳角、传感器、季节反射率、SAR 后向散射都可能改变影像外观，而地表真实状态未必改变。反过来，洪水、灾害、城市扩张和土地覆盖演化是真实地表状态变化，但其观测结果受成像条件影响。

因此，遥感世界模型的核心矛盾是：

**如何从有偏像素观测中分离地表状态，并建模地表状态在外生驱动下的演化。**

### 第三段：为什么普通视频世界模型不够

自然视频世界模型通常处理密集帧率、相机连续运动、物体交互和动作条件。遥感不同：重访稀疏、观测多源异构、成像物理强、外生驱动不可控、反事实配对样本缺失、地理物理先验可验证。

所以遥感需要自己的 world model 定义，而不是把视频模型搬过来。

### 第四段：本文定义

本文将遥感世界模型定义为：

**从遥感像素观测中估计地表状态，并学习该状态在外生驱动、地理物理先验和成像条件下如何转移与被观测的模型。**

形式化上，模型学习：

```text
P(ΔS, U, R | X历史, S当前, D外生驱动, G地理物理先验, Φ成像条件, h预测跨度)
```

其中 `ΔS` 是状态转移分布，`U` 是不确定性，`R` 是结构化解释或一致性证据。

### 第五段：本文方法与评估

提出一个统一框架，包含状态估计、驱动条件潜转移、地理物理一致性验证、分层不确定性和可选像素观测解码。评估不只看影像指标，而是看状态转移、驱动敏感性、驱动留出泛化、时间邻近自然对照、物理一致性和校准。

## 六、Abstract 逻辑骨架

建议 Abstract 按四步写：

1. **背景缺口**：现有遥感大模型主要建模观测，难以回答地表状态在外生驱动下如何演化。
2. **核心定义**：本文提出面向像素观测的遥感地表状态动力学世界模型，将世界定义为地表状态及其可观测投影。
3. **方法与基准**：模型从历史观测、当前状态、外生驱动和地理物理先验中预测未来状态转移分布，并用驱动敏感性和物理一致性协议评估。
4. **贡献结果**：在洪水、城市扩张、灾害损毁、土地覆盖演化等任务上证明，该模型比普通时序预测和生成式遥感模型更能响应驱动条件、更物理一致、更可校准。

## 七、Contribution 写法

建议贡献写成三条，不要超过四条。

### 贡献一：范式与定义

提出遥感语境下的地表状态动力学世界模型定义，明确区分地表状态与像素观测、外生驱动与智能体动作、干预式情景模拟与强因果反事实。

解决的问题：防止“没有 action 就不是 world model”的质疑，也防止“只是未来帧生成”的质疑。

### 贡献二：方法框架

提出一个面向像素观测的状态动力学框架：以多源历史观测估计当前状态，以外生驱动和地理物理先验调制潜状态转移，并输出状态转移分布、分层不确定性和一致性证据。

解决的问题：让“世界模型”落到模型结构，而不是停留在概念命名。

### 贡献三：评估协议与基准重构

将多个遥感地表演化任务重构为统一状态转移问题，并设计驱动条件留出、驱动敏感性、时间邻近自然对照和物理一致性评估。

解决的问题：证明模型真的使用外生驱动并遵守地表演化约束，而不是只在视觉指标上好看。

## 八、如何定义 research gap

当前 research gap 不应写成“遥感还没有世界模型”，因为已有 `Remote Sensing-Oriented World Model` 和 `RS-WorldModel`。更稳的写法是：

**已有遥感世界模型主要从空间外推、变化理解或未来场景生成角度定义世界模型，但尚未系统地把地表状态转移、外生驱动响应、成像观测偏置和可验证地理物理约束统一到同一个世界模型形式化中。**

这个 gap 更准确，也更难被一篇已有工作直接打掉。

## 九、如何解释“为什么遥感需要世界模型”

遥感需要世界模型，不是因为 world model 是热门词，而是因为遥感存在普通感知模型解决不了的四个结构性问题。

1. **观测有偏**：像素受云、太阳角、传感器、季节和分辨率影响，不能直接等同于地表状态。
2. **时间稀疏**：遥感不是高帧率视频，缺测和不规则重访是常态。
3. **变化由外生过程驱动**：洪水、灾害、气候、人类活动不是模型内部 action，但决定地表未来。
4. **合理性可验证**：地形、水系、道路、已有建筑、气候带、季节转移等先验可用于检验预测是否像一个“世界”。

这四点共同说明：遥感需要的不是更大的图像预测器，而是能把观测、状态、驱动和约束分开的世界模型。

## 十、如何避免被认为是“把视频世界模型搬到遥感”

必须在论文里主动切割。

| 容易被误解的点 | 推荐表述 |
|---|---|
| 视频模型预测未来帧，本文也预测未来 | 本文预测的是地表状态转移，未来影像只是可选观测投影。 |
| 视频世界模型依赖 action，遥感没有 action | 遥感里的条件是外生驱动和观测条件，不是智能体动作；这是科学世界模型而非 RL 世界模型。 |
| 视频评估看 FVD/FID，本文也可生成图 | 本文核心评估是驱动敏感性、状态转移准确性、约束一致性和不确定性校准。 |
| 视频数据密集连续，遥感数据稀疏不规则 | 本文把不规则重访、多源异构和成像条件作为遥感世界模型的结构性特征。 |
| 视频世界是相机看到的场景 | 本文的世界是地表状态，影像只是观测。 |

## 十一、最推荐的最终主线

最终建议采用：

**外生驱动条件下的遥感地表状态转移世界模型。**

但在方法和实验中保留像素观测：

**像素用于估计状态、监督状态、验证状态和可视化状态；状态动力学才是论文主角。**

这条线最符合 CVPR/AAAI 论文的审稿逻辑：问题真实、定义清楚、与已有工作可切割、方法细节能服务叙事、实验也能围绕“世界模型能力”组织。
