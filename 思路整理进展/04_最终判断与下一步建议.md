# 最终判断与下一步建议

## 一、是否建议作为 CVPR/AAAI 投稿方向

建议继续作为 CVPR/AAAI 投稿方向推进，但必须收缩主线。

当前方向不是“已经稳到可以直接开写完整论文”的状态，而是“有顶会潜力，但需要先完成叙事定型”的状态。最有希望的切口不是泛泛地说“遥感世界模型”，而是：

**外生驱动条件下的遥感地表状态转移世界模型。**

如果这条线能被严格实现和评估，它具备 CVPR/AAAI 所需要的问题意识：遥感观测有偏、状态演化由外部过程驱动、现有基础模型偏观测理解而非地表动力学、已有生成式世界模型缺少可检验状态转移和物理一致性评估。

## 二、最推荐采用的叙事主线

最推荐主线：

**从像素观测到地表状态动力学：外生驱动条件下的遥感世界模型。**

它的核心论断是：

遥感里的“世界”不是图像流，而是地表状态；遥感世界模型的任务不是生成一张好看的未来图，而是在给定外生驱动、地理物理先验和成像条件下，预测地表状态如何转移，并说明预测是否可信、是否符合地理物理规律。

## 三、最需要删除或弱化的内容

| 内容             | 建议         | 原因                                                 |
| -------------- | ---------- | -------------------------------------------------- |
| “像素空间世界模型”作为标题 | 弱化         | 容易被理解为未来帧生成，与 RS-WorldModel、TerraMind、UniTS 竞争正面冲突 |
| 主动获取作为第一主线     | 暂时删除或放未来工作 | 会把文章拉向主动采样、调度和不确定性决策，导致主线过宽                        |
| 大段自然语言解释/LLM 壳 | 弱化         | 不是世界模型核心，容易被看成遥感 VLM 附加功能                          |
| 从零训练大模型作为贡献    | 弱化         | 算力规模不是科学贡献，且会让审稿人期待大模型级别结果                         |
| 过强因果反事实表述      | 修改         | 观测遥感数据难以支持强因果识别，应改为操作性驱动替换和情景模拟                    |
| 过多任务一次性全做      | 收缩         | 任务太多会像工程整合，建议先保证 3 至 4 类状态转移任务讲清楚                  |

## 四、最需要强化的内容

| 内容 | 为什么必须强化 |
|---|---|
| 遥感世界模型定义 | 必须明确“世界 = 地表状态及其观测投影”，否则容易被认为概念包装 |
| 状态本体 | 必须说明状态类别、转移空间、像素级/对象级如何统一 |
| 外生驱动定义 | 必须区分驱动、任务结构、标签和成像条件，防止信息泄漏 |
| 成像观测与地表状态的分离 | 这是遥感区别于自然视频世界模型的关键 |
| 驱动敏感性评估 | 这是证明“不是普通时序预测”的命门 |
| 物理一致性验证 | 这是证明“世界模型”不是单纯生成模型的关键 |
| 与已有遥感世界模型的切割 | 必须正面比较 RS-WorldModel、RemoteBAGEL、Earth-o1、TerraMind |

## 五、下一步优先完成的 5 件事

### 1. 写一页“不可再摇摆”的问题定义

只写五个东西：世界是什么、状态是什么、驱动是什么、观测是什么、输出是什么。不要写模型结构。

推荐定义：

```text
世界 = 地表状态场
观测 = 该状态在特定传感器与成像条件下的像素投影
驱动 = 影响地表状态转移的外生过程
任务 = 预测未来状态转移分布
评估 = 状态准确性 + 驱动响应 + 物理一致性 + 不确定性校准
```

### 2. 做一张相关工作切割矩阵

必须至少包含：

`RS-WorldModel`、`Remote Sensing-Oriented World Model`、`Earth-o1`、`TerraMind`、`SkySense`、`Prithvi-EO-2.0`、`V-JEPA 2`、`DreamerV3`、`GraphCast`、`Aurora`。

矩阵字段建议为：世界定义、状态表示、条件类型、是否预测未来、是否显式建模地表状态、是否有驱动敏感性评估、是否有物理一致性评估。

### 3. 先做一个最小可证伪任务

不要一开始做完整大模型。建议先选洪水或城市扩张做 pilot：

1. 有 `D` 与无 `D`；
2. 真驱动、空驱动、错驱动；
3. 加 `G` 与不加 `G`；
4. 输出状态转移；
5. 报驱动敏感性和物理一致性。

如果这个 pilot 证明不了驱动真的有用，整篇世界模型叙事就要重构。

### 4. 固定像素与状态的关系

进入代码前必须定死：

像素是主输出，还是辅助输出？

我的建议是：

**主输出为状态转移；像素输出为辅助观测解码。**

这能最大限度降低生成赛道风险。

### 5. 先写 Introduction 骨架再写方法

先写中文 Introduction 骨架，确保每段只服务一个逻辑节点：

1. 现有遥感模型偏观测理解；
2. 遥感真正需要建模地表状态演化；
3. 外生驱动和观测偏置使遥感不同于视频世界模型；
4. 定义遥感地表状态动力学世界模型；
5. 提出方法和评估协议；
6. 总结贡献。

如果 Introduction 写不顺，说明方法还不该开工。

## 六、进入代码实现前必须想清楚的问题

1. `S_t` 的状态标签从哪里来？是语义分割、对象标签、外部产品，还是模型估计？
2. `D` 是否真的包含驱动信息，而不是把任务标签偷偷塞进去？
3. 哪些任务有明确外生驱动，哪些只是时间趋势？
4. 不同数据集的状态转移是否能共用一个本体，还是只能共享框架？
5. 像素解码失败是否会拖累主线？
6. 物理一致性验证器是规则、统计经验分布，还是学习模型？
7. 驱动敏感性没有真实反事实时，如何避免过度声称？
8. 与 RS-WorldModel 相比，实验上至少要赢在哪里？
9. 如果基础模型 baseline 很强，你的世界模型能力如何单独证明？
10. 第一版投稿是偏 CVPR 方法论文、AAAI 概念方法论文，还是 NeurIPS Datasets and Benchmarks 风格？

## 七、最终判断

当前想法可以成立，但成立的方式不是继续扩大，而是收缩。

最稳的判断是：

**继续做，但不要做“像素生成式遥感世界模型”；要做“像素观测约束的地表状态动力学世界模型”。**

如果后续实验能证明模型对外生驱动敏感、状态转移准确、物理约束更一致、不确定性更可信，那么这条线有 CVPR/AAAI 投稿价值。反过来，如果实验只能证明生成图像更像、或者多任务平均指标更高，世界模型叙事会显著变弱。

## 八、本次输出依据

本次判断综合了本地想法稿、文献笔记和外部调研。关键外部依据包括：

- [RS-WorldModel](https://arxiv.org/abs/2603.14941)：已有遥感世界模型命名与“理解变化 + 预测未来”主线。
- [Remote Sensing-Oriented World Model](https://arxiv.org/abs/2509.17808)：已有方向条件空间外推式遥感世界模型。
- [Earth-o1](https://arxiv.org/abs/2605.06337)：大气领域 observation-native 世界模型范例。
- [Vision World Models Survey](https://www.preprints.org/manuscript/202604.2072)：视觉世界模型三组件和超越外观评测。
- [Agentic World Modeling](https://arxiv.org/abs/2604.22748)：L1/L2/L3、法则域和干预敏感性评估框架。
- [SkySense](https://openaccess.thecvf.com/content/CVPR2024/html/Guo_SkySense_A_Multi-Modal_Remote_Sensing_Foundation_Model_Towards_Universal_Interpretation_CVPR_2024_paper.html)：遥感基础模型对比背景。
- [TerraMind](https://arxiv.org/html/2504.11171v1)、[Prithvi-EO-2.0](https://arxiv.org/abs/2412.02732)：遥感/地球观测基础模型背景。
