## 1. 既有世界模型机器盘点（file:line，全部已核实）

### 1.1 Partition 模型（composed 一致性的载体）
`models/dynamics/obsworld_partition.py`
- `class ObsWorldPartitionModel(ObsWorldRolloutModel)`，`forecast_mode = "obsworld_partition_24d"`。
- 主输出 = 普通递归 20×5 天 rollout（继承自 Rollout）。
- `_two_step_partition()`（55–163 行）：对同一条 D/C/Δt 路径，在一次 10 天直接转移 vs 一次 5+5 天组合转移之间，产出：
  - `z_direct`（139）、`z_mid`（140）、`z_composed`（141）——潜态；
  - `pred_direct`（142）、`pred_composed`（143）——解码像素（注意：`decode_states` **不传 baseline**，故为绝对解码）；
  - `partition_start` 由训练循环采样，`detach_partition_start=True`（88–92）只切断辅助分支梯度，不改主 rollout 的显存/训练预算。
- 转移接受**任意中间态**为第一参（`controlled_transition.py:62` forward 的 `state` 参数），`elapsed_days = Δt.sum`（107 行）按段求和 → **组合律良定义**：两跳的物理 elapsed 正确。

### 1.2 一致性 loss（无可学习参数，防退化）
`models/dynamics/partition_consistency.py`
- `class PartitionConsistencyLoss`（41 行）。四项加权（默认权重）：
  - `w_state=0.10`：对称 stop-grad 的 `z_direct≈z_composed`，用 `F.layer_norm(..., weight=None, bias=None)`（126–148）——**故意无可学习投影**，否则可学习层会把两条路径"假装对齐"而底层动力学仍不一致。
  - `w_observation=0.10`：`pred_direct≈pred_composed`（masked huber，150）。
  - `w_ndvi=0.05`：两支 NDVI 一致（151–155）。
  - `w_endpoint=0.50`：**两支都对真实 target 监督**（157–172）——这条保证了低 state_gap 不能靠退化成常数态骗到；也意味着 partition run **不是纯弱 rollout**，endpoint 项在管精度。
- 诊断量：`state_gap`、`state_std_direct/composed`（192–197），可直接进 Table 3。
- `from_config()`（80–101）读 `loss.partition.{state,observation,ndvi,direct_endpoint,endpoint_ndvi_weight}`。
- `sample_two_step_partition_start()`（20–38）：均匀采样合法 5+5 锚点；用全局 torch RNG（trainer 已 checkpoint），resume 可复现。

### 1.3 训练循环激活（已接线）
`train/train_stage2_earthnet.py`
- 67–68 行 import `PartitionConsistencyLoss`、`sample_two_step_partition_start`；84 行 import `partition_loss_scale`。
- 1648–1649：`partition_loss_fn = PartitionConsistencyLoss.from_config(...)`。
- 1812：`partition_scale = partition_loss_scale(config, optimizer_step)`。
- 1814：`partition_start = sample_two_step_partition_start(...)`（仅在 scale>0 时）。
- 1864–1868：调用 `partition_loss_fn(z_direct=..., z_composed=..., pred_direct=..., pred_composed=..., target=..., target_mask=...)`。
- 1881：`partition_losses["total"] * partition_scale` 加入总 loss。

### 1.4 curriculum 调度（纯函数，resume 安全）
`train/stage2_curriculum.py`
- `partition_loss_scale()`（134–170）：`training.partition` 必须是 mapping 且 `enabled=true`（155–159 否则报错，**防止静默跑主方法却不加一致性 loss**）；`optimizer_step<warmup_start_step` 返回 0；之后线性 ramp 到 1。
- `rollout_length_for_step()`（51–106）：`training.rollout_curriculum` 必须**从 step 0 起、长度非减**（98/93 行强制），防 resume 缩短开环步长。
- `PARTITION_FORECAST_MODES`（25–27）含 **`obsworld_partition_physical4`**。

### 1.5 工厂路由（v2 正式路径）
`models/dynamics/obsworld_factory.py`
- `create_obsworld_v2_model()`（54）：v2 协议（`earthnet2021x_path_v2`）走这里，**不走遗留 `ObsWorldStage2Model`**。
- `V2_PARTITION_MODES`（48–50）含 `obsworld_partition_physical4`；186–213 按 `mode` 选 wrapper：Direct/Rollout/Partition/Correction。
- Partition wrapper 与 Direct **共享同一 `core` + `transition`**（160–178），**不新增参数**——故 S1a→S1b 的 state_dict 兼容（见 §4 resume 选项）。

### 1.6 关键澄清（纠正我最初的设想）
- `compute_latent_targets`（`obsworld_stage2.py:38/155`）造 `z_target=encode(未来帧)` 的 latent-future loss，**仅遗留 `ObsWorldStage2Model` 用**（`train:497` 只喂给遗留 factory）。**v2 正式路径不用它。**
- → **S1b（v2）的潜态世界模型证据 = composed-consistency（`z_direct≈z_composed`），不是 `z_pred≈z_target`。** 别再往 v2 路径加 latent-future loss，那是另一条（遗留）路线。

## 2. S1b 的两个实现选项（**待你醒来定**）

### 选项 A（就绪、纯配置、安全）：既有 partition run
- config 已写好：`configs/train/plan_a_s1b_partition_physical4.yaml`（继承 Plan A recipe，切 `forecast_mode: obsworld_partition_physical4`）。
- 产出 Table 3：composed-consistency gap + 两支 endpoint 精度 + 供冻结读出复用的状态。
- **代价（诚实）**：partition 继承递归 rollout，其**头条 rollout 精度低于 S1a Direct**（误差累积）。**绝不把它当精度/SOTA 项**；Table 1 永远是 S1a Direct。
- 风险：低（纯配置，机器已验证接线）。

### 选项 B（需签字、要 ~40 行新代码、故事最强）：在 Direct 上加 compose 辅助
- 思路：给 `ObsWorldDirectPathModel` 加一个**默认关**的 compose 辅助分支（对某个 endpoint 走 5+5 组合），复用同一 `transition` 与 `PartitionConsistencyLoss`，**不切换到递归 rollout**。
- 产出：**同一个强 Direct 模型**既拿 SOTA 精度（Table 1）**又**具备 composed 一致性（Table 3）——"我们的强预报器的潜态本身就是可组合的充分统计量，加一致性正则不掉精度"。这是最强的"SOTA + 世界模型"合一叙事。
- 代价：需改 `obsworld_direct_path.py`（additive、config-gated、默认关，不影响正在跑的 S1a）+ 训练循环给 Direct 分支也走 partition_loss（现在 partition_loss 只在 partition mode 触发，见 `train:1853`）。需 smoke。
- 风险：中（新代码，需审查 + smoke），但**默认关 = 现有行为不变**，隔离性好。

**我的建议**：先跑**选项 A**拿到 Table 3 的最小可信证据（配置就绪、零代码风险）；若 S1a 精度确认漂亮、时间允许，再上**选项 B**做"合一"强化。两者不互斥。**但选项 B 动模型架构，属于"每步前审核"的一步，等你点头再写。**

## 3. S1b config 逐项（选项 A，已写入 `plan_a_s1b_partition_physical4.yaml`）

```yaml
_base_: plan_a_stage2v3_vits_train.yaml   # 继承 Stage1.5 强制 + physical4 + 全微调
model:
  forecast_mode: obsworld_partition_physical4
  decoder: {residual: false}              # partition 分支本就无 baseline，整跑绝对解码，单一语义
training:
  open_loop: true                         # rollout 必需
  teacher_forcing_future_state: false     # curriculum 禁止 teacher forcing
  rollout_curriculum:                     # 从 step0 起、长度非减
    - {start_step: 0,    length: 5}
    - {start_step: 1500, length: 10}
    - {start_step: 4000, length: 20}
  partition:
    enabled: true                         # 否则 partition_loss_scale 报错（防静默跑无 loss）
    warmup_start_step: 1500
    warmup_steps: 1500
    detach_partition_start: true
loss:
  partition: {state: 0.10, observation: 0.10, ndvi: 0.05, direct_endpoint: 0.50, endpoint_ndvi_weight: 0.50}
```
- `max_steps: 8800` 由 physical4 base 继承。**注意**：rollout 每步做至多 20 次顺序转移，wall-clock 比 Direct 慢；若 8800 步不够收敛，先看 loss 曲线再决定加步（别一上来堆步数）。

## 4. 备选：resume-from-S1a-best（更快的选项 A 变体）
- Direct 与 Partition 共享 `core/transition/decoder` 子模块名（`obsworld_factory.py` 同一批组件），故用 `resume_from: <S1a checkpoint_best>` 冷启 partition run，state_dict 兼容（partition wrapper 不新增参数）。
- 好处：从已训好的强 Direct 态出发，只学"让它可组合"，收敛更快、状态更好。
- 注意：resume 会带上 optimizer step；curriculum/partition warmup 的 step 语义要对齐（`rollout_curriculum` 必须从 0 起，故 resume 时 step 已>0，会直接落到 length=20 相位——这可能太激进）。**若走 resume，需把 curriculum 的 start_step 平移到 resume 起点附近，或干脆 length 直接给 20 并把 partition warmup 设短。** 这条比选项 A 主线复杂，等你定。

## 5. 上线判据与门禁（写死）

**门禁 G（S1a → S1b 的唯一开关）**：S1a 在 epoch100（step≈4400）的 val：
- 若 **RGBN-MAE 明显低于旧基线 0.0331 且 NDVI-MAE 低于 0.1087**（residual+finetune 抬起了精度）→ 值得为世界模型证据再花一次 8×H200 → 启动 S1b（选项 A）。
- 若精度没动 → **先别启动 S1b**；回到 §2 重想（可能先修 S1a，或直接走选项 B 的"合一"避免弱 rollout）。

**S1b 自身判据（跑起来后看什么）**：
- `state_gap` 随训练下降并收敛到小值（composed≈direct）→ 潜态可组合（Table 3 核心）。
- `endpoint_direct` 与 `endpoint_composed` 都保持低（两支都准）→ 一致性不是靠退化换来的。
- 若 `state_gap` 降但 endpoint 精度崩 → 说明在用退化态骗一致性，需调 `direct_endpoint` 权重。
