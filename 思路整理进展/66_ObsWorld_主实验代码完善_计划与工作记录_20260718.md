## 1. 本轮已完成：item 3 结果统计核心

**新增 `eval/stage2_result_statistics.py`（纯附加，不改已钉 commit 的 `greenearthnet_protocol.py`）**，三块能力：

1. `per_horizon_ndvi_metrics(target, prediction)` — 单 cube 的**逐时距（day 5,10,…,100）NDVI RMSE 与 R²**，返回可池化的充分统计量（Σ平方误差、像元数、Σ目标方差）。→ 供 **Figure 2**。
2. `aggregate_horizon_curves(frames)` — 把多 cube 的充分统计量**池化**成一条总体曲线（`rmse=√(ΣSE/n)`，`r2=1-ΣSE/Σtotvar`）。
3. `paired_per_cube_frame` + `tile_cluster_paired_bootstrap` — **配对、按 EarthNet tile 聚类的 bootstrap 95% CI**（对 tile 做有放回重采样，避免同 tile 内空间相关的 minicube 被当独立样本）→ 供 **Rollout-P4 vs Direct-P4 的中心论断**（对应 doc58 §4.6 强制的配对 CI，以及 §5.1 情形 A/B/C 判据）。

**验证**：`tests/test_stage2_result_statistics.py` **8 个单测全过**——完美预测→RMSE 0/R² 1；+0.1 偏移→RMSE 0.1；池化聚合与单 cube 一致；bootstrap 在"两模型相同"时 CI 含 0（不显著）、"一致更优 0.05"时 CI 排除 0（显著）；NaN 指标被丢弃不污染 CI；单 tile 抛错；tile 提取与空输入守卫。

**对抗式复核 + 修复（多轮审核第 2 轮）**：一个独立 agent 审出 10 条，已按严重度处理：
- **已修（真实 bug）**：① bootstrap 里 `point` 用 pandas.mean（跳 NaN）而重采样用 numpy.mean（传播 NaN）→ 只要一个 diff 为 NaN，CI 会**静默变 NaN**；现 `paired_per_cube_frame` 丢 NaN 行、bootstrap 前再 dropna，口径一致。② `n_clusters==1` 时 CI 退化成一点、**伪显著**；现要求 ≥2 tile 否则抛错。③ 池化 R² 原来是"每 cube 内居中"、漏了 cube 间方差；现携带 `Σtrue/Σtrue²/n`，池化时用**全局均值居中**（真·总体估计）。④ 逐时距 R² 是**空间解释方差**，与主表 R²（时序 Pearson r² 的 land-cover 平衡）是**两回事**；已改列名 `r2_spatial` 并在文档标注、建议**中心配对 CI 用 RMSE**（更稳）。⑦ `significant_95` 改名 `significant`（不再写死 95）。⑧ `p_crosses_zero`→`p_one_sided` 并对 point==0 守卫。⑨ 池化除零加 `np.errstate`。⑩ `n` 改为按非 NaN 交集计数，避免 rmse 被略微稀释。
- **仅记为口径说明（非 bug）**：⑤ 不等 tile 大小下比率估计的 O(1/K) 偏差 + 用的是百分位区间（tile 多时可忽略）；⑥ Figure2 曲线是**像元加权**、配对 CI 是**cube 加权**——两件主产物对"同一实验"的加权口径不同，属设计选择，已在模块 docstring 明示。

### 设计决策（**请你重点审这几条**）
- **D-3a 逐时距 R² 定义**：采用"在某 lead time t、跨合格像元的**空间解释方差**"`R²(t)=1-SS_res(t)/SS_tot(t)`（SS_tot 为该时刻真值 NDVI 的空间方差）。这是一种常见但非唯一的定义；若你要的是别的口径（如逐像元时序 R²、或 land-cover 平衡后的 R²），告诉我改。
- **D-3b 跨 cube 聚合 = 池化**（每合格像元等权），不是"每 cube 先算再平均"。若论文想 **land-cover 平衡**（与主表一致），我再加一层平衡。
- **D-3c bootstrap 聚类 = tile（cube id 前 5 位）**，`n_boot=2000`、95% CI。tile 数很少时（n_clusters 极小）CI 会退化——真实数据 tile 数很多，不是问题，但会在文档提示。

## 4. 需要你拍板的决策点（不阻塞我做 Persistence / item3，只挡 Climatology 定义）

- **D-1 Climatology/previous-year 基线定义**（挡 Outperformance 列）：
  - **已查准官方定义**（`vitusbenson/greenearthnet` 的 `model_pixelwise/climatology.py`）：**逐像元 day-of-year 气候态**——对 minicube 自身、**排除目标年**的其他年份，按 DOY 求均值（先线性插值、再 30 天 wrap 滚动平滑），评分时按最近 DOY 映射到目标时刻。**不是 previous-year，也不是逐 cube 上下文均值。**
  - **⚠️ 用真实数据核实到的硬冲突**：我们主表用的 **IID/OOD cube 是单年**（150 步 ~5 个月，如 2018-03-29→08-25）→ "排除目标年"后**无其他年可平均，官方气候态在 IID/OOD 上算不出**。只有 **seasonal split 是多年（1050 步，2017–2020），官方气候态才可算**。
  - **因此这是一个需要你拍板的论文口径决策**，选项：
    - (A) **Climatology 行/Outperformance 列只在 seasonal 报**（官方算法可忠实复现），IID/OOD 主表**不放 Climatology**、只放 Persistence（可算）——最诚实、无口径造假；
    - (B) 给 IID/OOD 造一个**替代气候态**（如训练集按 tile/land-cover×DOY 的均值），带 † 脚注声明偏离官方；
    - (C) 暂不报 Outperformance 列，等确定协议后再补。
  - 我的建议：**(A)**。它与 doc58/60 "跨协议只做量级参考、自研行走同协议官方评分" 的纪律一致；官方 climatology.py 我可**照搬实现**并只在 seasonal 上跑。
  - 来源：[GreenEarthNet 仓库 README](https://github.com/vitusbenson/greenearthnet/blob/main/README.md)、[climatology.py](https://raw.githubusercontent.com/vitusbenson/greenearthnet/main/model_pixelwise/climatology.py)、[CVPR2024 论文](https://arxiv.org/pdf/2303.16198v2)。
