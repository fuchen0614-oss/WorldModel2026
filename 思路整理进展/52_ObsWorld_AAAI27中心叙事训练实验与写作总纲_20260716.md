# 52. ObsWorld AAAI-27 中心叙事、主实验冻结与关键概念解释

> [!abstract] 本次修订解决什么
> 本文档不再把所有可做分支同时排进近期计划，而是回答：现在只做哪一个主实验、什么结果出来后冻结、U 到底是什么、Stage1.5 是否站得住、Direct 和 Partition 应怎样表述，以及论文的 3 图 3 表如何闭环。

> [!important] 一句话决策
> **先在同一 physical4 协议下完成并比较 Direct-P4 与 Rollout-P4，得到第一套完整主实验结果后冻结主干；full24、Partition（分段一致性）、U（观测更新）、下游任务和额外数据集都不得阻塞这一步。**

> [!warning] 与旧路线的关系
> 本文从 2026-07-17 起覆盖旧版 52 中“把 U 当成已经成立的中心贡献”和 57 中“下一步优先正式跑 full24”的建议。57 的工程审计仍有效，但实验优先级以本文为准。

## 0. 先给最直白的答案

### 0.1 当前唯一主干

~~~text
Stage1 表征预训练
      ↓
Stage1.5：φ 条件化的状态预训练（Stage2 使用中性 φ）
      ↓
Stage2-D：Direct-P4（严格配对的非递推对照）
      ↓ 同一数据、同一预算、同一 physical4
Stage2-R：Rollout-P4（共享五日状态转移的主模型）
      ↓
完整 val_dev 选 checkpoint → 冻结 → IID/OOD 正式评估
~~~

第一套“完整主实验”必须至少包含 Direct-P4 和 Rollout-P4 两行。只跑完 Direct 只能说明预测管线可用，不能说明共享状态递推有效；只跑 Rollout 又无法判断收益来自递推结构还是训练预算差异。

### 0.2 现在不做什么

- 不先跑 full24；它只是 driver-input ablation（驱动输入消融），不是主方法。
- 不先跑 Partition；它不能证明严格时间可组合性。
- 不扩展额外数据集；Stage2 固定为 raw NetCDF EarthNet2021x，并遵循 EarthNet2021 standard protocol。
- 不先做大模型 + ours、下游分类或额外迁移任务。
- 不在 Direct-P4 与 Rollout-P4 完成前并行开多个结构分支。

### 0.3 当前论文中心

在 U 尚未完成和验证前，工作标题使用：

> **ObsWorld: A Shared-Transition Predictive-State World Model for Long-Horizon Earth Observation Forecasting**<br>
> **ObsWorld：面向长期地球观测预测的共享转移预测状态世界模型**

它强调三件当前代码确实具备、且可以通过主实验验证的事情：

1. pretrained state initialization（预训练状态初始化）；
2. predictive state（预测状态）；
3. shared controlled rollout（共享受控递推）。

> [!critical] Stage2 的真实 φ 边界
> Stage1.5 预训练时真实 φ 同时调制编码器与重建解码器。raw EarthNet2021x 含时间、位置/瓦片、产品标识和质量掩码，但当前 Stage2 loader 没有把它们构造成与 Stage1.5 兼容且经验证的逐帧 φ；正式代码通过 make_neutral_s2_phi 使用固定 neutral φ（中性缺失条件），Stage2 的新观测解码器也不接收 φ。因此当前不能把“实时采集条件感知”作为 Stage2 主贡献，只能把 Stage1.5 称为 φ 条件化预训练，并用 Stage1 vs Stage1.5 的预测消融判断这些预训练权重是否有用。


只有当 U 完成端到端训练，并优于强在线更新基线后，标题才升级为：

> **ObsWorld: An Observation-Correctable Predictive-State World Model for Earth Observation Forecasting**<br>
> **ObsWorld：面向地球观测预测的可由观测校正的预测状态世界模型**

这不是降低目标，而是把已经实现的贡献与条件性升级贡献分开，防止摘要先承诺、实验却没有结果。

## 1. 对本轮 16 个问题的逐项结论

### 1.1 full24 真的重要吗

不重要到足以阻塞主实验。

physical4 使用四个有明确物理意义的五日驱动：降水总量、平均温度、平均 VPD（饱和水汽压亏缺）和太阳辐射总量；full24 则把八个日变量的 mean/min/max（均值/最小值/最大值）都输入模型。二者只回答：驱动表示用紧凑物理四维还是完整 24 维更合适。

当前决定：

- 正文主模型固定 physical4；
- full24 延后到主干冻结之后；
- 只有算力充足或审稿人明确追问时，才在 val_dev 做单 seed（单随机种子）附录对照；
- 如果完全不跑，就不能写“physical4 优于 full24”或“额外变量没有用”，只能写“本文采用预先规定的 physical4”。

### 1.2 各实验的精度、指标与 SOTA（当前最佳方法）能否定下来

可以冻结评价精度和成功规则，不能预先编造模型会达到的数值。

| 项目 | 冻结设置 |
|---|---|
| H200 训练 | BF16（脑浮点 16 位）混合精度；允许 TF32 矩阵乘；优化器状态保持 FP32 |
| 正式评价 | FP32 聚合；禁止用 BF16 累积最终指标 |
| R² / NSE / RMSE / 绝对 bias | 小数点后 3 位 |
| ENS / MAD / OLS / EMD / SSIM | 小数点后 4 位 |
| MAE / NDVI-MAE | 小数点后 4 位 |
| Outperformance（超越气候基线比例） | 1 位小数，单位 % |
| 参数量 / FLOPs | 0.1M / 0.1G |
| 速度 | 单 minicube 毫秒数，1 位小数 |

主表必须比较：

1. Persistence（持续性基线）；
2. Climatology（气候态基线）；
3. Contextformer；
4. PredRNN，若复现成本过高则用 SimVP，但二者至少一个；
5. Direct-P4；
6. Rollout-P4。

VegeDiff、EO-WM、VegSim 属于必须讨论的近期强相关方法，但输出形式、概率建模或评测轨道与当前 RGBN 主协议不完全一致。在没有同 manifest（清单）、同 mask（掩码）、同 evaluator（评估器）的重跑前，不能把论文数字混进严格主表。

成功规则：

- 主模型必须超过 Persistence 和 Climatology；
- Rollout-P4 在主要指标上不能相对 Direct-P4 明显退化；
- Rollout-P4 应在长时距或 OOD（分布外）中表现出至少一个稳定优势，否则“共享递推状态”的必要性不成立；
- Contextformer 的论文数字只是参考目标，不是跨协议硬门槛。

### 1.3 U 属于哪个阶段

U 属于 **Stage2-U（Stage2 的观测更新分支）**，不必另造一个模糊的 Stage3。

- Stage2-D：Direct（直接多时距预测）；
- Stage2-R：Rollout（共享状态递推）；
- Stage2-U：Update（获得新观测后的状态更新）；
- Stage2-P：Partition（可选分段一致性辅助分支）。

执行上先训练并冻结 Stage2-R，再训练一个小 U adapter（更新适配器）。第一轮冻结 I/F/O 主干，只训练更新器；U 已经有效后，才考虑用主干约 0.1 倍学习率联合微调。

### 1.4 Stage1.5 与非线性 φ 泄漏是否经得起推敲

当前训练策略在设计上经得起推敲，但不能声称“已经完全消除了非线性 φ 泄漏”。

当前 60k 配置在 Stage1.5 预训练阶段确实较完整：

- φ 经过独立 phi_encoder（采集条件编码器）；
- φ 从 ViT 第 8 层开始通过 FiLM（特征线性调制）进入 encoder（编码器）；
- φ 同时进入 decoder（解码器），让观测重建知道采集条件；
- S1/S2 七天内配对状态使用 VICReg 对齐；
- cross-covariance loss（交叉协方差损失）抑制状态与 nuisance（干扰条件）的线性相关；
- teacher anchor（教师锚定）限制状态漂移；
- state projector + reconstruction bridge（状态投影器与重建桥）形成可迁移的状态接口。

以上真实 φ 只存在于 Stage1.5 预训练。迁移到 EarthNet Stage2 后，代码加载 encoder、phi_encoder 与 state_projector 权重，但对每帧使用固定 neutral φ，并改用新的 EarthNet observation decoder（观测解码器）。因此合理目标是：**Stage1.5 预训练让状态保留未来预测信息，并在预训练数据上把已知采集条件交给 φ 分支解释；其是否改善 Stage2 必须由预测消融决定。** 这能鼓励去耦，但既不保证任意非线性 probe 都无法恢复 φ，也不代表 Stage2 正在使用真实采集元数据。

旧 MLP probe 的 67%–71% 不能评价当前模型：

1. 提取状态时既没有加载/传入 phi_encoder，也没有走 Stage2 的 make_neutral_s2_phi 中性条件路径，与两种真实路径都不一致；
2. Stage1 对照临时创建了随机初始化的 SpatialStateProjector；
3. 代码把第一个 patch 当作不存在的 CLS token（分类标记）丢掉；
4. probe 训练与评估没有严格独立的空间 tile（瓦片）切分；
5. 没有 metadata-only（仅元数据）基线，无法排除地点、季节与标签混杂。

所以 67%–71% 只保留为历史诊断，不能进入论文。

新的 φ 评估有用，但只属于附录/紧凑消融，不得阻塞主实验。最有价值的 Stage1.5 证据仍是：在相同 Rollout-P4 预算下比较 Stage1 初始化与 Stage1.5 初始化。

如时间允许，再做修正版 probe：

- 用真实部署路径提取 actual-φ（真实 φ）和 neutral-φ（中性 φ）状态；
- 按地理 tile 划分 probe train/test；
- 比较 metadata-only、state-only、metadata+state 和 shuffled-state（打乱状态）；
- 报告 metadata+state 相对 metadata-only 的增量 ΔR² / ΔAUROC；
- 同时使用 linear ridge（线性岭回归）和固定小 MLP，5 个 seed；
- 报告 majority baseline（多数类基线）、balanced accuracy（平衡准确率）与 AUROC；
- 固定图像只改变 φ，检查状态漂移和解码可控性。

最终用词：

- 安全默认：acquisition-conditioned state pretraining transferred with a neutral condition（以中性条件迁移的采集条件化状态预训练）；
- 若新评估成立：reduces incremental nuisance predictability beyond metadata（降低元数据之外的增量干扰可预测性）；
- 禁止：removes all nonlinear acquisition leakage（完全去除所有非线性采集条件泄漏）。

### 1.5 Direct 是“工程地基”会不会贬低模型

会，所以删除这种写法。论文中称为：

> **matched non-recursive control（严格配对的非递推对照）**

Direct 与 Rollout 使用相同的初始化、physical4、解码器、数据、预算和监督目标。区别仅在于：

- Direct 从同一初始状态分别预测各未来时距；
- Rollout 复用同一短步转移，把上一步状态继续推进。

Direct 很强反而让证据更可信。若 Rollout 优于 Direct，说明共享状态推进有价值；若二者相当，说明递推没有付出明显性能代价；若 Direct 明显更好，就必须承认当前共享转移尚未学稳。

### 1.6 Partition 是什么，能证明时间可组合性吗

Partition（分段一致性）可以用开车比喻：同一段十天驱动路径，一次走十天，与先走五天再走五天，最终状态是否接近。

~~~text
z0 --同一段 10 天 D/C/Δt--> z_direct

z0 --前 5 天--> z_mid --后 5 天--> z_composed
~~~

它只能支持“同一控制路径下，10 天与 5+5 天分段近似一致”。它不能证明严格 semigroup（半群）、任意时间尺度可组合、真实物理定律、因果机制或更高预测精度。

当前决定：Partition 不进入第一套主实验，甚至可以完全不做。只有 Rollout 已成功且附录还有预算时，才做单 seed；若 gap（差距）变小但预测变差，也不算成功。

### 1.7 英文词怎样处理

第一次出现统一写成“英文（中文）”，例如：

- predictive state（预测状态）；
- open-loop rollout（开放循环递推）；
- observation correction（观测校正）；
- out-of-distribution, OOD（分布外）。

终稿附录另有术语表，避免只堆英文缩写。

### 1.8 Observation Correction 到底是什么，和不确定性是不是一回事

不是一回事。

- uncertainty（不确定性）回答：“模型对未来有多没把握？”通常输出方差、分布或多个未来。
- observation correction（观测校正）回答：“现实中途给了一张新的、部分清晰的观测，怎样修改内部状态，再继续预测？”

U 的顺序：

~~~text
先预测第 k 步 → 新观测到达 → 只看有效区域残差
→ 更新状态 → 继续预测 k+1...
~~~

它可以把质量或不确定性作为门控输入，但不等于不确定性模块。当前代码已接通 correction model、模型工厂、trainer reveal 日程和基础 evaluator；但尚未完成“从冻结 Rollout 安全初始化、冻结 I/F/O 只训练 U、固定 day25/day50 reveal、Gain-AUC/配对 CI 与 no-reveal 一致性”的正式协议，因此仍然没有可进入论文的 U checkpoint 和结果。

### 1.9 是否应该先完整做一个主实验再冻结

应该。但“一个主实验”不是只跑一个 checkpoint，而是：

1. Direct-P4 seed 42；
2. Rollout-P4 seed 42；
3. 完整 val_dev 选 checkpoint；
4. 冻结配置与选择规则；
5. IID/OOD 各评估一次；
6. 形成 Table 1 与 Figure 2 第一版。

闭环成功后，再补两个 seed 得到 mean±std（均值±标准差）。此前不跑 full24、Partition、U 或下游。

### 1.10 数据集暂时怎样写

正文准确写为：Stage2 使用 **raw NetCDF EarthNet2021x v1.0**，遵循 **EarthNet2021 standard protocol（标准协议）** 的 10→20 帧任务和 train/IID/OOD/extreme/seasonal 五划分；内部 val_dev 只用于 checkpoint 选择。Stage1/1.5 使用 SSL4EO-S12 v1.1 做表征预训练。这是标准的 pretrain-then-adapt（预训练后适配），不矛盾。

正式出数仍须冻结原始 bundle、manifest hash（清单哈希）、mask 和 evaluator 版本，防止数据名、样本划分与评分口径混淆。

### 1.11 为什么改成“预测状态世界模型”，U 又是条件升级

当前能由代码和结果检验的是：历史观测初始化状态、共享转移推进状态、解码为未来观测。U 尚无端到端实现。

而且 2026 年 EO-WM 和 VegSim 已明确使用 weather-driven world model（天气驱动世界模型）或 geospatial world model（地理空间世界模型）定位。因此只写“外生天气驱动世界模型”已经不够独特。当前更稳妥的差异点是：

- φ 条件化预训练得到的状态权重以中性条件迁移到 Stage2；
- 同一观测空间中可验证的共享短步递推；
- U 成功后，再增加真实再观测后的状态更新。

“条件升级”直白地说就是：**U 做成并赢了，标题和摘要升级；U 没做成，当前主论文仍能成立，但不能写可观测校正。**

### 1.12 “证据与决策门矩阵”是什么意思

它不是愿望表，而是：每一句想写进论文的大话，提前指定哪个实验允许我们写；若实验失败，提前规定删掉或缩小哪一句。

| 想写的主张 | 需要的证据 | 若失败怎么办 |
|---|---|---|
| 共享状态递推有效 | Rollout-P4 对 Direct-P4 的长时距/OOD 结果 | 删除“递推更优”，先修模型 |
| Stage1.5 有用 | 同预算 Stage1 vs Stage1.5 | 降为训练细节 |
| U 能校正状态 | U 优于 no-update、restart、VanillaFilter | 不升级标题，移到未来工作 |
| Partition 支持分段一致 | gap 下降且预测不退化 | 删除实验与主张 |

### 1.13 physical4 应怎样优先

不是“physical4 先跑、full24 随后立刻跑”，而是“整个第一套主实验只用 physical4”：

~~~text
Direct-P4 → Rollout-P4 → 冻结 → 正式结果
~~~

full24 在冻结后重新判断是否值得做，不自动进入下一步。

### 1.14 3 图 3 表为何有 U/no-U 两种版本

这不是同时写两篇论文，也不是训练分叉。它只是防止把正文位置绑定到尚未实现的模块。

当前执行 no-U（无 U）版本。如果 U 后来通过机制门，只替换 Table 2 和 Figure 3，标题与摘要再升级；其余主结果不变。

### 1.15 本文怎样修订 52

旧版用 I-F-H-U，容易造成 U 已在正式训练的误解。本版改为：

- I-F-O 是当前可执行主干；
- U 是 Stage2-R 冻结后的条件升级；
- 用 O 表示 observation decoder（观测解码器），避免 H 与 horizon（预测时距）冲突；
- 用唯一 physical4 主干替代多分支并行。

完整中文论文终稿见 [[58_ObsWorld_AAAI27中文论文终稿_主实验冻结版_20260717]]。之后直接修改 58，不再为每轮讨论新增“终稿”文件。

## 2. 当前论文的科学问题与差异化

### 2.1 任务

给定 50 天历史多光谱地球观测，以及未来 100 天内的 prescribed drivers（已知外生驱动），预测未来地表观测序列。

这里模拟的是 Earth-observable surface dynamics（可由卫星观测验证的地表演化），不是完整地球系统，也不是数字孪生。

### 2.2 不能再单独使用的创新表述

- “第一次把天气用于遥感预测”；
- “第一次把地球观测预测称为世界模型”；
- “第一次递推 NDVI”；
- “第一次考虑未来不确定性”。

Contextformer 已系统使用气象条件；VegeDiff 做概率扩散预测；EO-WM 和 VegSim 已提出天气驱动或情景驱动的地球观测世界模型。

### 2.3 当前可守住的贡献

在 U 之前：

1. **问题组织**：把长期预测组织为 pretrained predictive state（预训练预测状态）的初始化、共享推进与观测渲染。
2. **方法**：从跨模态预训练状态出发，在紧凑物理驱动下复用共享五日转移，开放循环递推 100 天。
3. **证据**：用严格配对 Direct、时距曲线、OOD 和驱动扰动验证状态递推是否有用。

若 U 成功，再增加：

4. **观测更新**：通过可见区域残差更新内部状态，并改善再观测后的预测。

## 4. 方法记号：当前是 I–F–O，U 条件接入

历史观测与采集条件形成初始预测状态：

$$
z_0=I_\theta(x_{1:T_c};\bar\phi),\qquad \bar\phi\text{ 为固定中性条件}.
$$

共享状态转移推进五日状态：

$$
z_{k+1}=F_\theta(z_k,D_k,C_k,G,\Delta t_k).
$$

观测解码器还原未来 RGBN：

$$
\hat x_{k+1}=O_\theta(z_{k+1}).
$$

- I：initializer（初始化器）；
- F：controlled transition（受控状态转移）；
- O：observation decoder（观测解码器）；
- D：physical4 五日驱动；
- C：calendar（历法/季节信息）；
- G：geography（静态地理与地形）；
- bar-φ：Stage2 固定 neutral acquisition condition（中性采集条件参考）；真实逐帧 φ 只用于 Stage1.5 预训练，不是 Stage2 活跃输入。

U 成功后，得到部分有效观测时才增加：

$$
z_{k+1}^{+}=U_\psi\left(z_{k+1}^{-},
E(x_{k+1})-E(\hat x_{k+1}),q_{k+1},a_{k+1}\right).
$$

q 是可见质量，a 是观测年龄。没有观测或 q=0 时必须严格满足：

$$
U(z,\cdot,0,a)=z.
$$

## 6. Stage2-U：主干冻结后如何训练

第一版 reveal schedule（观测揭示日程）：

- 50% 样本无中途观测；
- 50% 样本恰好一次观测；
- reveal step 在未来第 2–15 个五日步中采样；
- 必须先预测第 k 步，再把真实观测给 U；
- 更新损失只看 reveal 之后的未来步。

U 接收更新前状态、观测特征减预测特征的残差、有效可见比例 q、staleness（观测陈旧度）与 reveal mask。

必须比较：

| 基线 | 直白含义 |
|---|---|
| No-update | 看到了新图也不用 |
| Re-encode / restart | 用新图重新初始化 |
| Naive latent blend | 固定比例混合新旧状态 |
| VanillaFilter / generic GRU | 参数量匹配的通用更新器 |
| ObsWorld-U | 质量与残差门控更新 |

评价 day 25 和 day 50，报告更新后的绝对误差、paired gain（配对收益）、Gain-AUC（收益曲线面积）、清晰度分层，以及无观测路径是否退化。

只有 U 优于 No-update、Restart、Naive blend 和容量匹配 Filter，且 q=0 严格不变，才进入标题与摘要。

## 7. Stage1.5 的最小论文证据

Stage1.5 承担 I 的初始化质量，不是可随意丢弃的支线。最有价值的证据不是旧 probe，而是：

| 初始化 | Rollout-P4 IID | Rollout-P4 OOD | 长时距误差 | 收敛速度 |
|---|---:|---:|---:|---:|
| Stage1 | 待填 | 待填 | 待填 | 待填 |
| Stage1.5 | 待填 | 待填 | 待填 | 待填 |

- 若预测稳定改善：Stage1.5 是方法贡献；
- 若只加速收敛：写成 optimization aid（优化辅助）；
- 若无改善：降为实现细节。

完全无法预测 φ 不一定合理，因为轨道、季节、地点和真实外观可能相关。正确问题是：在已知元数据之外，状态是否还增加了无预测价值的采集干扰可预测性。

## 8. Direct、Rollout、Partition 的逻辑

~~~text
Direct-P4
  └─ 不递推，只从 z0 预测各未来时距，能做到多好？

Rollout-P4
  └─ 同一个短步 F 反复推进，能否维持长期预测状态？

Partition-P4（可选）
  └─ 同一 10 天路径一次走完与 5+5 走完是否近似一致？
~~~

- Direct 成功：可写“长期预测器”；
- Rollout 不弱且长时距更好：可写“共享递推预测状态”；
- Partition gap 下降：只能写“分段一致性得到支持”；
- U 成功：才可写“可由新观测校正”。

## 14. 能说与不能说

可以说：

- 在给定历史观测、未来外生驱动和静态地理背景下预测可观测地表演化；
- 学习并递推一个可解码、可由真实未来观测检验的预测状态；
- Stage1.5 显式处理采集条件；
- 若结果成立，共享短步转移改善或保持长期预测；
- 若 U 成立，新观测能修正后续预测。

暂时不能说：

- 模拟完整真实地球或数字孪生；
- 学到严格因果动力学；
- physical4 是唯一真实物理因素；
- 完全去除非线性 φ 泄漏；
- Partition 证明严格半群；
- U 已正式训练；
- 仅凭跨协议数字达到 SOTA。

## 15. 中英术语表

| 英文 | 中文直白解释 |
|---|---|
| acquisition condition, φ | 采集条件，如太阳角度、轨道、卫星平台 |
| acquisition-conditioned pretraining | 预训练时用真实采集条件；当前 Stage2 只迁移权重并使用中性条件 |
| predictive state | 为预测未来而维护的内部状态 |
| open-loop rollout | 中途不喂真实未来，依靠自身状态连续递推 |
| matched control | 除被研究因素外其余设置完全一致 |
| exogenous driver | 模型外部给定的驱动，如天气 |
| horizon | 从当前到预测目标的时间距离 |
| IID / OOD | 分布内 / 分布外 |
| observation correction | 得到新观测后更新内部状态 |
| uncertainty | 对预测有多没把握，不等于观测更新 |
| reveal | 在某个未来时点把真实观测提供给模型 |
| no-op | 不改变状态 |
| staleness | 距上次有效观测过去多久 |
| ablation | 去掉或替换一个组件，看它是否有用 |
| provenance | 数据、代码、配置、模型来源记录 |
| bootstrap confidence interval | 重复抽样估计结果差异的不确定范围 |
