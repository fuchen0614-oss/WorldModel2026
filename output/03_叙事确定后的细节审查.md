# 叙事确定后的细节审查

## 一、已确定的主线

后续所有细节都应服务于以下主线：

**外生驱动条件下的遥感地表状态转移世界模型。**

为了兼顾像素建模偏好，可在副标题或正文中表述为：

**面向像素观测的遥感地表状态动力学世界模型。**

这里的边界很重要：像素是观测界面，状态转移是世界模型核心。

## 二、输入输出定义

### 输入

| 输入                 | 含义                          | 是否必要 | 叙事作用               |
| ------------------ | --------------------------- | ---- | ------------------ |
| 历史像素观测 `X_{t-k:t}` | 光学、SAR、多光谱或其他遥感影像序列         | 必要   | 提供地表状态的观测证据        |
| 当前状态先验 `S_t`       | 水体/非水体、建筑、土地覆盖、作物、损毁前状态等    | 强烈建议 | 把论文从影像预测转成状态转移     |
| 外生驱动 `D`           | 洪水、灾害类型、降雨强度、城市扩张强度、季节转换等   | 必要   | 支撑世界模型的条件响应能力      |
| 地理物理先验 `G`         | DEM、坡度、距河流、距道路、永久水体、气候带、月份等 | 必要   | 支撑遥感独特性与物理一致性      |
| 成像条件 `Φ`           | 传感器、太阳角、视角、云、极化、时间戳等        | 建议   | 支撑“像素观测不等于地表状态”的叙事 |
| 预测跨度 `h`           | 未来时间间隔或事件后间隔                | 必要   | 让任务成为动力学推演而非静态理解   |

### 输出

| 输出 | 含义 | 建议优先级 |
|---|---|---|
| `ΔS_{t:t+h}` | 未来地表状态转移分布 | 最高 |
| `U` | 分层不确定性，区分数据模糊与模型未知 | 高 |
| `R` | 结构化解释或物理一致性证据 | 中高 |
| `X_{t+h}` 或 `Ŷ_{t+h}` | 可选未来像素观测或重建影像 | 中 |

建议不要把未来像素作为唯一主输出。最稳的设计是：主输出为状态转移，像素输出作为辅助解码、可视化和观测一致性约束。

## 三、任务定义

主任务应定义为：

**给定历史像素观测、当前地表状态、外生驱动、地理物理先验和预测跨度，预测未来地表状态转移分布，并估计该预测的不确定性与物理一致性。**

不要定义为：

**给定过去影像，生成未来影像。**

后者会让论文直接落入遥感视频预测和生成模型赛道。

## 四、训练目标

建议训练目标由四类组成，不要一开始堆太多。

| 目标 | 作用 | 是否必须 |
|---|---|---|
| 状态转移监督损失 | 监督 `S_t -> S_{t+h}` 的转移类别或连续状态变化 | 必须 |
| 观测一致性损失 | 让潜状态能解释像素观测，可用重建、掩码重建或观测解码 | 建议 |
| 驱动条件对比/敏感性损失 | 防止模型忽略 `D`，鼓励真实驱动和空驱动/错驱动产生可区分响应 | 建议 |
| 物理一致性或验证器损失 | 对洪水低洼、建筑邻域、土地覆盖转移等约束进行软惩罚 | 建议 |
| 不确定性校准损失 | 使模型置信度与真实错误率对齐 | 建议 |
| 自然语言解释损失 | 只在结构化解释足够稳后加入 | 可选 |

最小可投稿版本不需要一口气完成所有训练项，但必须证明驱动条件和物理约束真的起作用。

## 五、像素空间建模是否合理

结论：**不建议把“像素空间世界模型”作为核心叙事；建议采用“像素观测 + 状态动力学联合建模”。**

原因如下。

| 判断点 | 结论 |
|---|---|
| 像素空间建模能否增强叙事 | 能，但只能作为观测约束和可视化，不应取代状态转移。 |
| 像素预测是否足以证明世界模型能力 | 不足。清晰图像可能只是纹理生成，不代表学到驱动下的地表状态规律。 |
| 像素建模风险 | 预测模糊、成本高、容易被 FID/SSIM 牵着走，且要面对 RS-WorldModel、TerraMind、UniTS 等生成类工作。 |
| 像素建模的正确位置 | 用来重建当前观测、解码未来状态、辅助跨传感器一致性验证。 |
| 最推荐表述 | “面向像素观测的地表状态动力学世界模型”，而不是“像素空间世界模型”。 |

一个可行折中是：模型内部在潜状态空间做动力学，输出端提供状态图和可选像素观测图。论文指标以状态转移和驱动响应为主，像素指标为辅。

## 六、是否需要潜在状态

需要。

如果没有潜在状态，论文会退化为“多条件遥感预测模型”。潜在状态至少承担三件事：

1. 把云、太阳角、传感器差异等观测偏置与地表真实状态分开；
2. 让动力学发生在“地表状态”而不是“像素亮度”上；
3. 使模型能够输出不确定性和结构化解释。

但潜在状态不一定必须是完全不可解释的黑箱向量。建议采用“语义状态 + 连续潜状态”双层结构：

| 状态层 | 内容 | 作用 |
|---|---|---|
| 显式语义状态 | 水体、建筑、土地覆盖、损毁等级、作物类型等 | 可监督、可评估、可解释 |
| 连续潜状态 | 纹理、湿度、植被活力、隐含驱动响应等 | 保留像素与物理过程中的细粒度信息 |

这样既满足世界模型对潜状态的需求，又不牺牲遥感任务的可检验性。

## 七、方法设计方向

建议方法模块收敛为六块。

| 模块 | 功能 | 叙事必要性 |
|---|---|---|
| 观测编码器 | 从历史像素观测中提取多源时空特征 | 必要 |
| 状态估计器 | 将观测转为当前地表状态信念 | 必要 |
| 驱动编码器 | 编码洪水、灾害、季节、城市扩张等外生驱动 | 必要 |
| 地理物理先验编码器 | 注入 DEM、水系、道路、气候、月份等先验 | 必要 |
| 状态转移算子 | 在潜状态/语义状态上预测未来转移 | 核心 |
| 一致性与不确定性头 | 输出物理合理性、不确定性和解释 | 建议 |

可选加入观测解码器，将未来状态或当前状态渲染回像素观测。但它应作为辅助模块，不应成为论文最重的部分。

## 八、数据集需求

数据集选择必须服务于外生驱动状态转移主线。

| 任务 | 数据集候选 | 支撑的世界模型能力 | 风险 |
|---|---|---|---|
| 洪水状态转移 | Sen1Floods11、SEN12-FLOOD、外接 DEM/JRC 水体 | 灾害驱动、地形约束、事件响应 | 前后时相配对不一定完整，需要补历史观测 |
| 城市扩张 | SpaceNet 7 | 慢变量、多主体人类活动驱动、长期状态转移 | 驱动条件较弱，容易被看成建筑变化检测 |
| 灾害损毁 | xBD | 事件冲击、对象级状态转移、灾害类型留出 | 单步、非连续时序，需明确是 shock-response |
| 土地覆盖演化 | DynamicEarthNet | 季节驱动、多类语义状态转移 | Planet 许可、标签频率与影像频率不一致 |

建议第一版不要超过 3 至 4 类任务。若时间有限，优先选择洪水、城市扩张、土地覆盖，再把 xBD 作为对象级扩展。

## 九、实验设计

实验不应只是“比谁指标高”，而应证明世界模型能力。

| 实验 | 证明的问题 |
|---|---|
| 主状态转移预测 | 模型能否预测地表状态如何变化 |
| 驱动条件消融 | 去掉或打乱 `D` 后性能是否下降，证明驱动真的有用 |
| 驱动敏感性分析 | 固定观测和状态，替换真实/空/错误驱动，预测分布是否合理变化 |
| 驱动留出泛化 | 留出某类灾害、区域或季节，测试模型是否学到可泛化驱动语义 |
| 地理物理一致性评估 | 洪水是否偏向低洼/近水体，建筑是否靠近道路/已有建筑，土地覆盖转移是否合理 |
| 时间邻近自然对照 | 同一 AOI 的驱动期与非驱动期预测是否能区分 |
| 不确定性校准 | 高不确定区域是否对应错误、边界、缺测或分布外样本 |
| 像素辅助解码消融 | 像素观测约束是否提升状态预测，而不是只提升图像外观 |

## 十、对比方法

对比方法应覆盖四类。

| 对比类别 | 代表方法或方向 | 对比目的 |
|---|---|---|
| 遥感时序/变化检测模型 | ConvLSTM、PredRNN、Earthformer、变化检测专用模型、UniTS 等 | 证明不是普通时序预测 |
| 遥感基础模型 | SkySense、Prithvi-EO-2.0、RobSense 等 | 证明不是普通表征迁移 |
| 遥感生成/世界模型 | RS-WorldModel、RemoteBAGEL、TerraMind 可用部分 | 证明差异在状态转移与驱动响应 |
| 任务专家模型 | 洪水、建筑变化、xBD、土地覆盖任务 SOTA | 证明实际性能不弱 |

如果部分模型无法公平复现，可用公开报告或冻结骨干 + 统一任务头作为近似对比，但论文中必须说清楚。

## 十一、消融实验

| 消融 | 目的 |
|---|---|
| 无外生驱动 `D` | 证明不是只靠历史影像外推 |
| 错误驱动/空驱动 | 证明模型对驱动方向敏感 |
| 无地理物理先验 `G` | 证明物理约束不是装饰 |
| 无状态估计，只做像素预测 | 证明状态建模优于未来帧生成 |
| 无观测条件 `Φ` | 证明成像偏置分离有用 |
| 无不确定性校准 | 证明模型不仅预测，还知道何时不可信 |
| 无跨任务状态本体 | 证明统一建模不是任务拼接 |

## 十二、评价指标

建议分为四组。

| 指标组 | 可用指标 | 作用 |
|---|---|---|
| 状态转移准确性 | F1、IoU、mIoU、对象级 F1、转移混淆矩阵 | 主性能 |
| 驱动响应 | KL 散度、真实驱动与空驱动概率差、错驱动下主转移衰减 | 世界模型能力 |
| 物理一致性 | 坡度/水系一致性、道路邻近一致性、转移矩阵合理性、违规率 | 遥感独特性 |
| 不确定性 | ECE、AUSE、错误-不确定性相关、拒答覆盖精度 | 可信性 |
| 像素观测辅助 | SSIM、LPIPS、PSNR、FID | 只作辅助，不作主指标 |

## 十三、风险与修改建议

| 风险 | 严重程度 | 修改建议 |
|---|---|---|
| 被看成 RS-WorldModel 的变体 | 高 | 把未来场景生成降为辅助，主打状态转移、驱动响应和物理一致性 |
| 被看成普通时序预测 | 高 | 必做驱动敏感性、驱动留出和物理约束评估 |
| 像素空间建模过难 | 高 | 内部做潜状态动力学，像素作为观测解码辅助 |
| 数据集异质性过强 | 中高 | 先做少量任务，统一状态本体和 driver schema 后再扩展 |
| `do(D)` 因果表述过强 | 中高 | 改为“操作性驱动替换”或“干预式情景模拟”，不声称识别真实因果效应 |
| 主动获取分散主线 | 中 | 第一篇弱化为未来能力，当前只保留信息增益作为附录或讨论 |
| 从零训练成本过高且不构成创新 | 中 | 可以自训骨干，但不要把“从零训练大模型”写成贡献 |

## 十四、细节审查结论

细节层面最重要的调整是：

**不要让像素生成统治论文。应让状态转移统治论文，让像素观测服务状态建模。**

只要这一点定住，方法、数据、实验就能统一。否则，文章会在“生成模型、时序预测、基础模型、主动感知、世界模型”之间来回摇摆，审稿人很难抓住贡献。
