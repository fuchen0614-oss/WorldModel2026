旧方案的问题是**目标冲突**。

```
Encoder(X, phi) → z
```

一边用 FiLM/cross-attention 把 `phi` 喂给 encoder，意思是“请利用成像条件”；另一边又用 counterfactual loss 要求“把 `phi` 打乱后，`z` 还不能变”。这就双标了。

结果只有两种：

```
FiLM 真有用 → phi 一换 z 就变 → 被 loss 惩罚
loss 满足 → phi 怎么换 z 都不变 → FiLM 被模型无视
```

所以旧方案训出来不好解释：到底是 `phi` 起作用，还是模型学会忽略 `phi`？

改进方案 A 更清楚：

```
Encoder: X → z
Decoder: (z, phi) → X_hat
```

encoder 不吃 `phi`，负责提取成像无关的地表状态；decoder 吃 `phi`，负责把同一个状态渲染成不同成像条件下的观测。

一句话：

> 旧方案让 `phi` 既要被利用、又不能影响 latent，逻辑打架；方案 A 把 `phi` 放到 decoder，让它只控制观测外观，state 更干净，论文也更好解释。
