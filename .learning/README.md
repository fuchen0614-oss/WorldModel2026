## 🚀 快速开始

### 1. 第一次使用
1. 打开 [大纲.md](大纲.md) 浏览整个学习地图
2. 确认你想从哪个阶段开始(建议从第一阶段顺序学习)
3. 对话中说 **"学习模式"** 或 **"进入学习模式"** 开始学习

### 2. 日常学习流程
```
你: 学习模式,我想学习 Stage1 的编码器
AI: [读取进度] → [确认前置知识] → [开始讲解]
    - 代码定位
    - 实现细节
    - 原理解析
    - 行业对比
    - 实践建议
你: [提问/要求深入某个细节]
AI: [针对性讲解]
你: 我理解了,继续
AI: [下一部分] → [总结] → 是否更新进度?
```

### 3. 记录笔记
- 每学完一个模块,可以基于模板填写笔记
- AI 会帮你生成笔记框架,你负责填充"用自己的话总结"的部分
- 笔记模板已经在 `笔记/` 目录下,边学边填

### 4. 管理疑问
- 遇到不懂的,立刻记录到 [疑问记录.md](疑问记录.md)
- 标记疑问级别(🔴 阻塞 / 🟡 影响完整性 / 🟢 好奇探索)
- 定期集中解决,解决后移到"已解决"区域

## 🎯 学习模式 vs 开发模式

### 开发模式(默认)
适合日常工作:推进项目、修 bug、优化代码。
- 简洁高效
- 快速定位问题
- 给出解决方案

**示例**:
```
你: Stage1.5 的 loss 突然 NaN 了
AI: [快速诊断] 看日志... 是 phi 字段有 inf,修复方案是...
```

### 学习模式
适合深度学习:理解原理、掌握实现、学习规范。
- 耐心讲解
- 代码+原理并重
- 循序渐进
- 鼓励提问

**示例**:
```
你: 学习模式,为什么 Stage1.5 用 FiLM 而不是 cross-attention?
AI: 好问题!让我们从条件注入的几种方式说起...
    [对比 FiLM / cross-attention / AdaLN]
    [解释 Stage1.5 的设计考量]
    [指向代码实现]
    想看具体的数学推导吗?
```

### 如何切换?
- **进入学习模式**: 说 "学习模式" / "讲解XXX" / "我想理解XXX"
- **退出学习模式**: 说 "退出学习模式" / "回到开发模式" / 或直接提开发任务

## 📚 核心文件说明

### [大纲.md](大纲.md)
**作用**: 整个项目的学习地图
**内容**:
- 5个阶段,从基础到进阶
- 每个阶段的学习目标、代码入口、原理要点、行业规范
- 进度标记(⬜ 未开始 / 🟨 进行中 / ✅ 已掌握)
- 已知坑点的引用

**使用建议**:
- 开始学习前先通读一遍,建立全局认知
- 学习过程中随时回来查"我在哪个位置"
- 每完成一个模块,手动更新进度标记(或让 AI 帮你)

---

### [进度追踪.md](进度追踪.md)
**作用**: 动态记录当前学习状态
**内容**:
- 当前正在学习的模块
- 各阶段的完成度
- 已掌握的概念列表
- 待解决的疑问
- 学习日志(时间线)

**使用建议**:
- 每次学习模式开始时,AI 会先读这个文件确认进度
- 学完一个模块,更新"已掌握概念"
- 新对话开始时,这个文件帮助 AI 快速同步状态

**重要**: 这是"跨对话记忆"的关键,如果你换了一个对话框,只要说"学习模式",AI 就会读这个文件知道你学到哪了。

---

### [疑问记录.md](疑问记录.md)
**作用**: 集中管理学习中的问题
**内容**:
- 待解决的疑问(按日期和模块组织)
- 已解决的疑问(留作复习)
- 每个疑问的详细描述、解决方案、相关资源

**使用建议**:
- 遇到不懂的立刻记录,不要"先放着,回头再说"(容易忘)
- 标记疑问级别,优先解决 🔴 阻塞性问题
- 已解决的疑问保留,以后可能还会遇到类似的

**格式示例**:
```markdown
### [2026-07-10] 为什么 VAE 的 latent 是正态分布?
**所属模块**: Stage1 VAE
**疑问级别**: 🟡 影响完整性
**问题详情**: ...
**解决方案**: (学习后填写)
```

---

### [笔记/](笔记/)
**作用**: 每个模块的详细学习笔记
**内容**:
- 已提供模板: `Stage1-VAE编码解码器.md`

**使用建议**:
- 模板提供了标准结构,但内容需要你自己填充

## 💡 学习建议

### 1. 循序渐进
- 严格按大纲顺序,每个阶段都打好基础再进入下一阶段
- 前置知识不牢,后面的内容会越来越吃力

### 2. 代码先行
- 不要只看文档和原理,一定要看代码
- 边看边运行,修改参数观察效果
- 不懂的函数,读源码或查文档

### 3. 主动提问
- 看到不懂的立刻问,不要"先放着"
- 问"为什么"比问"是什么"更重要
- 多问"如果改成XXX会怎样"

### 4. 记录笔记
- 笔记是写给未来自己的,不是写给 AI 的
- 用自己的话总结,确保真的理解了
- 代码 + 原理 + 实验,三者缺一不可

### 5. 动手实验
- 每个模块学完,跑一遍相关代码
- 做消融实验:去掉某个组件,观察影响
- 可视化:画图、打 log、看中间结果

### 6. 定期复习
- 一周后回顾笔记,看是否还记得
- 遇到新概念,联系旧知识
- 疑问记录里的"已解决"定期重看

## ❓ 常见问题

### Q: 我可以跳过某个阶段吗?
A: 不建议。大纲的顺序是按依赖关系设计的,跳过会导致后面的内容理解困难。如果确实想先了解某个模块,可以让 AI 快速补充前置知识。

### Q: 学习模式和开发模式可以混用吗?
A: 完全可以!比如开发时遇到不懂的原理,随时可以问"为什么",AI 会切换到讲解风格。关键是你的提问方式决定了 AI 的响应深度。

### Q: 笔记一定要自己写吗?
A: 强烈建议自己写。AI 可以帮你生成框架和代码摘录,但"用自己的话总结"这部分必须你来,这是检验理解程度的最好方式。

### Q: 新对话时 AI 会记得我的进度吗?
A: 会!只要你说"学习模式"或"继续学习",AI 就会读取 [进度追踪.md](进度追踪.md) 来同步状态。但前提是你在上一次学习结束时更新了进度。

### Q: 如果我暂停学习很久,重新开始怎么办?
A: 打开 [进度追踪.md](进度追踪.md) 看上次学到哪了,然后说"学习模式,从XXX继续"。建议先花10分钟复习上次的笔记。
